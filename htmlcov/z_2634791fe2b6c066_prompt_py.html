1"""Load the built-in routing agent's prompt and runtime config from ``router.md``. 

2 

3The router's system prompt and its runtime config (``provider``, ``model``, 

4``thinking_effort``, ``history_turns``) live together in a single bundled 

5Markdown file, :file:`router.md`, beside this module — YAML front matter for 

6the config, Markdown body for the prompt. This mirrors how user-defined agents 

7are authored (``agents/*.md`` parsed by 

8:func:`calfcord.agents.definition.parse_agent_md`), while keeping 

9the router file **bundled in the package** rather than user-managed: the router 

10is project infrastructure, and shipping the file inside the wheel/image (like 

11:file:`agents/memory_prompt.md`) means a deploy can never lose its router 

12config. 

13 

14The prompt instructs the LLM to answer one question for every ambient 

15(``kind="message"``) Discord message: **"who is the user talking to?"** The 

16LLM picks one agent from the listed roster and emits the answer as a single 

17``<tool_name>(...)`` structured-output call (default tool name: 

18:data:`ROUTER_OUTPUT_TOOL_NAME`) carrying the agent id plus a short reasoning 

19string. The "exactly one agent" policy is enforced two ways: at the schema 

20level (:attr:`RoutingDecision.agent_id` is a single string, not a list) and at 

21the prompt level (the LLM is told to pick one addressee and that the chosen 

22agent can pull in peers out-of-band via its ``private_chat`` tool — the router 

23itself never fans out). 

24 

25The per-call agent roster is injected via ``temp_instructions`` (built by 

26:func:`calfcord.router.roster.build_router_temp_instructions`) 

27rather than baked into the file, so a newly-added agent becomes visible to the 

28router on the very next invocation without a restart. 

29 

30Interpolation 

31------------- 

32The body references the structured-output tool name 

33(:data:`ROUTER_OUTPUT_TOOL_NAME`) and the :class:`RoutingDecision` field names 

34(``agent_id``, ``reasoning``). These are **not** hardcoded in the Markdown: the 

35body carries ``{{ROUTER_OUTPUT_TOOL}}`` / ``{{AGENT_ID_FIELD}}`` / 

36``{{REASONING_FIELD}}`` placeholders that :func:`load_router_md` substitutes 

37from the code constants at load time, so renaming the tool or a field stays a 

38one-edit change in code. The substitution is a plain :meth:`str.replace` 

39(mirrors :mod:`calfcord.agents.memory`'s ``{{MEMORY_DIR}}`` 

40handling), and a surviving ``{{...}}`` after substitution is treated as a 

41typo'd placeholder and fails fast. A coupling test 

42(``tests/router/test_prompt.py``) confirms the names referenced in the rendered 

43prompt match the schema, and the field-name constants below are asserted 

44against :class:`RoutingDecision` at import time. 

45 

46Override 

47-------- 

48The bundled file is read by default; ``CALFKIT_ROUTER_PROMPT_PATH`` points the 

49loader at a mounted file to override the whole thing (config front matter + 

50prompt body) without rebuilding the image. The parsed config and rendered 

51prompt are cached on a module global; an operator editing the file takes effect 

52on the next router restart. 

53""" 

54

92_AGENT_ID_PLACEHOLDER = "{{AGENT_ID_FIELD}}" 

93_REASONING_PLACEHOLDER = "{{REASONING_FIELD}}"

98def load_router_md() -> tuple[RouterConfig, str]: 

99 """Return ``(config, system_prompt)`` parsed from ``router.md``, cached. 

100 

101 Reads ``CALFKIT_ROUTER_PROMPT_PATH`` when set, otherwise the bundled 

102 :file:`router.md` beside this module. The YAML front matter is validated 

103 into a :class:`RouterConfig` (so ``extra="forbid"`` rejects any 

104 router-identity field), and the Markdown body has its 

105 ``{{ROUTER_OUTPUT_TOOL}}`` / ``{{AGENT_ID_FIELD}}`` / ``{{REASONING_FIELD}}`` 

106 placeholders substituted from the code constants. Cached on a module 

107 global; an operator editing the file takes effect on the next restart. 

108 

109 Raises: 

110 ValueError: if the file cannot be read (e.g. a configured override 

111 path that doesn't exist), has no YAML front matter (missing / 

112 mistyped ``---`` fences), its front matter is malformed or invalid 

113 (unknown / reserved key, bad enum, out-of-range ``history_turns``), 

114 the body is empty, or a ``{{...}}`` placeholder survives 

115 substitution (a typo'd placeholder that would otherwise reach the 

116 LLM verbatim). The path is included in the message so the error is 

117 self-describing in container logs. 

118 """ 

119 global _cached 

120 if _cached is not None: 

121 return _cached 

122 

123 override = os.getenv(_PROMPT_PATH_ENV) 

124 path = Path(override).expanduser() if override else _DEFAULT_PROMPT_PATH 

125 

126 try: 

127 post = frontmatter.load(path) 

128 except (OSError, UnicodeError) as e: 

129 # ``UnicodeError`` (covers ``UnicodeDecodeError``) is a ``ValueError`` 

130 # subclass, not an ``OSError`` — a readable-but-non-UTF-8 override file 

131 # would otherwise escape as a bare, context-less ``ValueError``. 

132 raise ValueError( 

133 f"cannot read router prompt at {path} ({_PROMPT_PATH_ENV}={override!r}): {e}" 

134 ) from e 

135 except yaml.YAMLError as e: 

136 # ``frontmatter.load`` lets ``yaml.YAMLError`` propagate unchanged; 

137 # re-raise as ``ValueError`` with the path so the malformed-front-matter 

138 # path is self-describing and callers that catch ``ValueError`` cover it. 

139 raise ValueError(f"{path}: malformed YAML frontmatter: {e}") from e 

140 

141 metadata = dict(post.metadata) 

142 if not metadata: 

143 # No (or unclosed / BOM-prefixed / mistyped) ``---`` fences: frontmatter 

144 # returns empty metadata and folds the whole file — config lines 

145 # included — into the body. Fail loudly (mirrors ``parse_agent_md``) 

146 # rather than silently booting on all-default config with the raw config 

147 # text leaked into the prompt. Every bundled/override router.md must 

148 # carry front matter. 

149 raise ValueError( 

150 f"{path}: no YAML front matter found ({_PROMPT_PATH_ENV}={override!r}); " 

151 f"the router config (provider/model/thinking_effort/history_turns) must " 

152 f"appear between '---' fences at the top of the file" 

153 ) 

154 

155 try: 

156 config = RouterConfig(**metadata) 

157 except ValidationError as e: 

158 # Wrap the pydantic error with the file path so operators see which 

159 # file the validation failed against. Pydantic's own message lists 

160 # each invalid field with its location. 

161 raise ValueError(f"{path}: invalid router config: {e}") from e 

162 

163 _cached = (config, _render_body(post.content, path)) 

164 return _cached

167def _render_body(raw: str, path: Path) -> str: 

168 """Substitute the body placeholders and validate the result. 

169 

170 Raises: 

171 ValueError: if the body is empty after substitution, or a ``{{...}}`` 

172 placeholder survives (only the three known placeholders are 

173 expected, so any remaining brace pair is a typo). 

174 """ 

175 body = ( 

176 raw.replace(_TOOL_PLACEHOLDER, ROUTER_OUTPUT_TOOL_NAME) 

177 .replace(_AGENT_ID_PLACEHOLDER, _AGENT_ID_FIELD) 

178 .replace(_REASONING_PLACEHOLDER, _REASONING_FIELD) 

179 .strip() 

180 ) 

181 if not body: 

182 # An empty body makes the router prompt-less and useless; fail loudly 

183 # rather than ship an agent with no instructions. 

184 raise ValueError(f"{path}: router prompt body is empty") 

185 if "{{" in body or "}}" in body: 

186 raise ValueError( 

187 f"{path}: unsubstituted placeholder remains in router prompt body " 

188 f"(expected only {_TOOL_PLACEHOLDER}, {_AGENT_ID_PLACEHOLDER}, " 

189 f"{_REASONING_PLACEHOLDER})" 

190 ) 

191 return body 

192 

193 

194# Eager binding so ``from ...router.prompt import SYSTEM_PROMPT`` keeps working 

195# and a malformed bundled ``router.md`` fails fast at import (mirrors the old 

196# import-time assertions). ``load_router_md`` caches, so this single read is 

197# reused by ``build_router_definition``. 

198SYSTEM_PROMPT = load_router_md()[1]
