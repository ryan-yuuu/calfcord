1"""Agent identity, runtime hints, and system prompt parsed from ``agents/*.md``. 

2 

3Each agent is declared as a single Markdown file with YAML frontmatter and 

4a system-prompt body. The frontmatter declares identity and intrinsic 

5runtime hints; the body is the system prompt fed to the LLM. 

6 

7Format (matches Claude Code's ``.claude/agents/*.md`` convention):: 

8 

9 --- 

10 name: scheduler 

11 display_name: "Aksel (Scheduler)" 

12 description: "Calendar mechanics; book and prep meetings" 

13 provider: anthropic 

14 model: claude-sonnet-4-5 

15 tools: [calendar, email] 

16 thinking_effort: high 

17 --- 

18 

19 You are Aksel, the Scheduler. ... 

20 

21When ``avatar_url`` is omitted, :func:`parse_agent_md` fills it with a 

22DiceBear "glass" URL seeded by the agent's name so every assistant gets 

23a stable, recognizable persona avatar without operators having to host 

24images. Set ``avatar_url`` explicitly in the frontmatter to override. 

25 

26The YAML key is ``name`` (Claude Code parity). Internally the field is 

27``agent_id`` via a Pydantic alias so existing ``spec.agent_id`` access 

28patterns are preserved across the codebase. The Discord slash command for 

29each agent is always ``/<name>``; there is no separate ``slash`` 

30frontmatter field. 

31 

32``thinking_effort`` is the one frontmatter field that is operator-tunable 

33at runtime — the ``/thinking-effort`` Discord slash command rewrites it 

34via :mod:`calfcord.agents.md_writer`. Channel subscriptions 

35and other strictly deployment-specific state still live outside the .md 

36(see :mod:`calfcord.agents.state`). 

37""" 

38

52Provider = Literal["anthropic", "openai", "openai-codex"] 

53"""Supported LLM provider tags for the ``provider`` frontmatter field. 

54 

55The factory maps each provider to a concrete model-client class: 

56 - ``"anthropic"`` → :class:`calfkit.AnthropicModelClient` 

57 - ``"openai"`` → :class:`calfkit.OpenAIModelClient` 

58 - ``"openai-codex"`` → 

59 :class:`calfcord.providers.codex.CodexSubscriptionModelClient` 

60 (routes through ChatGPT Plus/Pro subscription billing rather than 

61 OpenAI API credits; requires ``uv run calfkit-auth codex login``). 

62""" 

63 

64ThinkingEffort = Literal["none", "minimal", "low", "medium", "high", "xhigh", "max"] 

65"""Operator-facing thinking-effort tiers. 

66 

67Seven abstract levels mapped to provider-specific reasoning/thinking 

68parameters in :mod:`calfcord.agents.thinking`. Tier names 

69parallel Claude Code's effort vocabulary; ``minimal`` is the lightest 

70non-zero step (a hair above ``none``); ``xhigh`` is a calfkit-specific 

71step between ``high`` and ``max``. 

72"""

74AgentRole = Literal["assistant", "router"] 

75"""Agent role — distinguishes ordinary assistant agents from the built-in 

76routing agent. 

77 

78``"assistant"`` (default) is what every user-defined ``agents/*.md`` file 

79produces. ``"router"`` is reserved for the singleton built-in router 

80agent constructed by :func:`calfcord.router.definition.build_router_definition`; 

81the factory wires routers differently (single-topic subscription, no 

82standard gates, ``ToolOutput`` final-output type, explicit 

83``publish_topic``). 

84"""

87class AgentDefinition(BaseModel): 

88 """One agent's declarative definition: identity, runtime hints, and system prompt. 

89 

90 Validators mirror the constraints Discord imposes on slash commands and 

91 webhook usernames so misconfiguration fails at load time rather than at 

92 first invocation. The Discord slash command name is always derived as 

93 ``/<agent_id>`` — the ``agent_id`` validator already enforces the 

94 Discord-compatible ``[a-z0-9_-]{1,32}`` shape. 

95 """ 

96 

97 # ``extra="forbid"`` surfaces frontmatter typos (``provder: openai``, 

98 # ``thiking_effort: high``) at parse time rather than silently dropping 

99 # them — important now that a slash command can rewrite the same file. 

100 # It also catches stale ``slash:`` lines left behind from the removal 

101 # of the dedicated slash field. 

102 model_config = ConfigDict(frozen=True, populate_by_name=True, extra="forbid") 

103 

104 agent_id: str = Field(..., alias="name") 

105 display_name: str 

106 description: str 

107 avatar_url: str | None = None 

108 """Webhook persona avatar URL. ``None`` here means "use the Discord 

109 webhook's default avatar"; for .md-loaded agents, 

110 :func:`parse_agent_md` substitutes the per-agent DiceBear default 

111 when the frontmatter omits or nulls the field, so live assistant 

112 definitions read from disk always carry a concrete URL.""" 

113 provider: Provider | None = None 

114 model: str | None = None 

115 tools: tuple[str, ...] | None = None 

116 """Tools available to this agent's LLM, resolved against 

117 :data:`calfcord.tools.TOOL_REGISTRY`. 

118 

119 Semantics: 

120 - ``tools:`` omitted from frontmatter (default ``None``) — agent 

121 gets every registered builtin tool. Convenient default for 

122 general-purpose assistant agents. 

123 - ``tools: []`` in frontmatter — agent gets NO tools (text-only). 

124 Explicit opt-out for read-only or routing-style agents. 

125 - ``tools: [a, b]`` — agent gets exactly those tools. 

126 

127 Security note: the "all by default" behaviour means a new 

128 ``agents/<name>.md`` ships with ``terminal`` / ``execute_code`` / 

129 ``write_file`` / ``patch`` access to the workspace bind-mounted into 

130 the ``calfkit-tools`` container unless the operator narrows the list 

131 explicitly. If you need a restricted-tools agent, add the 

132 ``tools:`` line. See :doc:`docs/authoring-agents` for the security 

133 model. 

134 

135 Router agents must omit ``tools:`` entirely (or set ``tools: []``); 

136 the validator rejects routers that declare non-empty tools.""" 

137 thinking_effort: ThinkingEffort | None = None 

138 role: AgentRole = "assistant" 

139 """Agent role. Defaults to ``"assistant"`` — ordinary user-defined 

140 agent. ``"router"`` is reserved for the singleton built-in routing 

141 agent; the factory uses a different wiring path for routers (no 

142 standard gates, single-topic subscription, ``ToolOutput`` final 

143 output type). User-authored ``agents/*.md`` should not set this 

144 field; the validator does not forbid it but operators wiring a 

145 second router will hit a registry boot error in :class:`AgentRegistry`.""" 

146 publish_topic: str | None = Field(default=None, min_length=1) 

147 """Optional explicit Kafka publish topic for the agent's 

148 ``ReturnCall``. Used by routers to declare their structured-output 

149 destination (where the fan-out consumer subscribes). ``None`` for 

150 assistant agents — they emit ``ReturnCall`` to the inbound frame's 

151 ``callback_topic`` (i.e., the bridge's ``discord.outbox``), which 

152 is the standard calfkit dispatch pattern.""" 

153 history_turns: int = Field(default=30, ge=0, le=100) 

154 """Number of recent channel messages the bridge fetches and projects 

155 into ``message_history`` on every invocation of this agent. 

156 

157 - ``0`` disables history fetching for this agent entirely (no 

158 Discord REST call; agent runs with the system prompt + user 

159 prompt only). 

160 - The upper bound (100) is Discord's per-call REST cap for 

161 ``channel.history(limit=...)``; raising it would force 

162 pagination, which is out of scope for v1. 

163 - The default (30) is a reasonable balance between context quality 

164 and token cost: ~30 messages of ~100 tokens average is ~3K 

165 input tokens per invocation, which is trivial on small models 

166 and acceptable on larger ones. 

167 

168 Set in ``.md`` frontmatter:: 

169 

170 --- 

171 name: scribe 

172 ... 

173 history_turns: 30 

174 --- 

175 

176 The router's analogous knob is ``history_turns`` in the bundled 

177 ``router.md`` front matter (see 

178 :mod:`calfcord.router.prompt`).""" 

179 memory: bool = False 

180 """Opt in to a persistent per-agent notepad. When ``True``, the factory 

181 registers a runtime instructions hook 

182 (:func:`calfcord.agents.memory.memory_instructions`) that 

183 appends the memory-explanation block to this agent's instructions on every 

184 invocation — the stored ``system_prompt`` is left unchanged. The block text 

185 is read only by the bridge (from the editable ``memory_prompt.md``, 

186 overridable via ``CALFCORD_MEMORY_PROMPT_PATH``) and shipped to the agent in 

187 ``deps``; the hook localizes it to ``memory/<agent_id>/`` in the shared 

188 workspace, where the agent keeps one-fact-per-file memories plus a 

189 ``MEMORY.md`` index, managed with the ordinary filesystem tools. 

190 

191 Default ``False`` so existing agents are unchanged. A memory-enabled agent 

192 must have the ``read_file`` and ``write_file`` tools — the factory enforces 

193 this at build time. See :doc:`docs/authoring-agents`.""" 

194 system_prompt: str 

195 source_path: Path | None = Field(default=None, exclude=True, repr=False) 

196 """Path to the ``.md`` file this definition was parsed from. Set by 

197 :func:`parse_agent_md`; ``None`` for in-memory test constructions 

198 and for the built-in router (which is constructed in code, not 

199 parsed from disk). ``exclude=True`` prevents accidental round-trip 

200 into a YAML dump; ``repr=False`` keeps logs tidy. Required for the 

201 ``/thinking-effort`` slash command's rewrite.""" 

202 

203 @field_validator("agent_id") 

204 @classmethod 

205 def _validate_agent_id(cls, v: str) -> str: 

206 if not AGENT_ID_PATTERN.fullmatch(v): 

207 raise ValueError(f"name must match [a-z0-9_-]{{1,32}}, got {v!r}") 

208 return v 

209 

210 @field_validator("display_name") 

211 @classmethod 

212 def _validate_display_name(cls, v: str) -> str: 

213 if not (1 <= len(v) <= 80): 

214 raise ValueError(f"display_name must be 1-80 chars, got {len(v)}") 

215 if v.lower() == "clyde": 

216 raise ValueError("display_name 'Clyde' is rejected by Discord webhooks") 

217 return v 

218 

219 @field_validator("description") 

220 @classmethod 

221 def _validate_description(cls, v: str) -> str: 

222 if not (1 <= len(v) <= 100): 

223 raise ValueError(f"description must be 1-100 chars (Discord slash limit), got {len(v)}") 

224 return v 

225 

226 @field_validator("system_prompt") 

227 @classmethod 

228 def _validate_system_prompt(cls, v: str) -> str: 

229 if not v.strip(): 

230 raise ValueError("system_prompt (markdown body) must be non-empty") 

231 return v

233 @field_validator("tools") 

234 @classmethod 

235 def _validate_tools(cls, v: tuple[str, ...] | None) -> tuple[str, ...] | None: 

236 """Syntax-check ``mcp/...`` tool entries; let bare builtin names through. 

237 

238 An ``mcp/...`` entry must be a well-formed selector 

239 (``mcp/<server>`` or ``mcp/<server>/<tool>``); all malformed entries 

240 are aggregated into ONE :class:`ValueError` naming each offending 

241 string, so an operator fixes the whole ``tools:`` line in a single 

242 pass. Whether the named server is configured or running is a 

243 *runtime* concern (the capability view resolves selectors per turn), 

244 deliberately not checked here — this is the authoritative syntactic 

245 gate for every read path: the agent process parses each ``.md`` 

246 through here before booting. 

247 

248 Bare names (anything not starting with ``mcp/`` — ``terminal``, 

249 ``calendar``, …) pass through **untouched**. Whether a bare name 

250 actually resolves to a registered builtin is checked later, by 

251 :meth:`AgentFactory._resolve_tools` against the factory's tool registry 

252 (defaults to :data:`~calfcord.tools.TOOL_REGISTRY`); rejecting 

253 arbitrary bare names here would couple this leaf schema to the tool 

254 registry and break code-built definitions that name tools the local 

255 registry doesn't (yet) hold. 

256 """ 

257 if v is None: 

258 return v 

259 bad: list[str] = [] 

260 for entry in v: 

261 if not is_mcp_selector(entry): 

262 continue 

263 try: 

264 validate_mcp_selector(entry) 

265 except ValueError as exc: 

266 bad.append(f"{entry!r}: {exc}") 

267 if bad: 

268 raise ValueError("malformed MCP tool selector(s) in tools: " + "; ".join(bad)) 

269 return v

271 @model_validator(mode="after") 

272 def _validate_router_constraints(self) -> AgentDefinition: 

273 """Enforce role-specific invariants on ``tools`` and ``publish_topic``. 

274 

275 Routers: 

276 - must declare no ``tools`` (the router uses pydantic-ai's 

277 ``ToolOutput`` pattern, where the "tool" is a 

278 schema-providing pseudo-tool whose args ARE the output — 

279 the body never runs; declaring real function tools 

280 alongside would muddy the LLM's tool list and the 

281 factory's wiring) 

282 - must declare a ``publish_topic`` (the fan-out consumer 

283 subscribes there; without it the router has no 

284 downstream consumer pathway) 

285 

286 Assistants: 

287 - must NOT declare a ``publish_topic`` (assistants emit 

288 ``ReturnCall`` to the inbound frame's ``callback_topic``, 

289 not to a fixed published topic; setting one would be a 

290 silent no-op that an operator might mistake for working 

291 custom-output wiring). 

292 """ 

293 if self.role == "router": 

294 if self.tools: 

295 raise ValueError( 

296 f"router agent {self.agent_id!r} must declare no tools; " 

297 f"got tools={list(self.tools)!r}" 

298 ) 

299 if not self.publish_topic: 

300 raise ValueError( 

301 f"router agent {self.agent_id!r} must declare a " 

302 f"publish_topic; the fan-out consumer subscribes there" 

303 ) 

304 else: # role == "assistant" 

305 if self.publish_topic is not None: 

306 raise ValueError( 

307 f"agent {self.agent_id!r} has role='assistant' but " 

308 f"declares publish_topic={self.publish_topic!r}; " 

309 f"publish_topic is reserved for routers — " 

310 f"assistants emit ReturnCall to the inbound " 

311 f"frame's callback_topic (set by the caller)" 

312 ) 

313 return self

316def parse_agent_md(path: Path) -> AgentDefinition: 

317 """Parse one ``agents/<name>.md`` file into an :class:`AgentDefinition`. 

318 

319 Enforces ``path.stem == frontmatter["name"]`` so the on-disk identifier 

320 and the declared name cannot drift. 

321 

322 Raises: 

323 FileNotFoundError: if ``path`` does not exist. 

324 ValueError: if the file has no frontmatter, the YAML is malformed, 

325 the name does not match the filename stem, or any field fails 

326 validation. 

327 """ 

328 try: 

329 post = frontmatter.load(path) 

330 except yaml.YAMLError as e: 

331 # ``frontmatter.load`` lets ``yaml.YAMLError`` propagate unchanged, 

332 # which would escape callers that catch only ``ValueError``. Re-raise 

333 # as ``ValueError`` so the docstring's contract holds and the 

334 # malformed-YAML path is indistinguishable from a malformed-name 

335 # path at the caller's seam. 

336 raise ValueError(f"{path}: malformed YAML frontmatter: {e}") from e 

337 metadata = dict(post.metadata) 

338 body = post.content 

339 

340 if not metadata: 

341 raise ValueError(f"{path}: missing YAML frontmatter") 

342 

343 declared_name = metadata.get("name") 

344 expected_name = path.stem 

345 if declared_name != expected_name: 

346 raise ValueError( 

347 f"{path}: frontmatter name={declared_name!r} does not match filename stem {expected_name!r}" 

348 ) 

349 

350 metadata["system_prompt"] = body.strip() 

351 # Resolve so the path remains valid even if the process later 

352 # ``os.chdir``s — the bridge daemon doesn't today, but a future 

353 # plugin or signal handler could. 

354 metadata["source_path"] = path.resolve() 

355 # Fill the per-agent DiceBear default when the .md omits avatar_url 

356 # (or sets it to ``null``). Done here at the load boundary rather 

357 # than as an AgentDefinition validator so code-built definitions — 

358 # the router, test fixtures, state-event projections — keep their 

359 # explicit ``None`` semantics. 

360 if metadata.get("avatar_url") is None: 

361 metadata["avatar_url"] = dicebear_avatar_url(declared_name) 

362 return AgentDefinition(**metadata)
