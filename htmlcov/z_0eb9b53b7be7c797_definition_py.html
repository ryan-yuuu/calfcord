1"""Agent identity, runtime hints, and system prompt parsed from ``agents/*.md``. 

2 

3Each agent is declared as a single Markdown file with YAML frontmatter and 

4a system-prompt body. The frontmatter declares identity and intrinsic 

5runtime hints; the body is the system prompt fed to the LLM. 

6 

7Format (matches Claude Code's ``.claude/agents/*.md`` convention):: 

8 

9 --- 

10 name: scheduler 

11 description: "Calendar mechanics; book and prep meetings" 

12 provider: anthropic 

13 model: claude-sonnet-4-5 

14 tools: [calendar, email] 

15 thinking_effort: high 

16 --- 

17 

18 You are Aksel, the Scheduler. ... 

19 

20The YAML key is ``name`` (Claude Code parity). Internally the field is 

21``agent_id`` via a Pydantic alias so existing ``spec.agent_id`` access 

22patterns are preserved across the codebase. The Discord slash command for 

23each agent is always ``/<name>``; there is no separate ``slash`` 

24frontmatter field. 

25 

26``thinking_effort`` is the one frontmatter field that is operator-tunable 

27at runtime — the ``/thinking-effort`` Discord slash command rewrites it 

28via :mod:`calfcord.agents.md_writer`. 

29""" 

30

43Provider = Literal["anthropic", "openai", "openai-codex"] 

44"""Supported LLM provider tags for the ``provider`` frontmatter field. 

45 

46The factory maps each provider to a concrete model-client class: 

47 - ``"anthropic"`` → :class:`calfkit.AnthropicModelClient` 

48 - ``"openai"`` → :class:`calfkit.OpenAIModelClient` 

49 - ``"openai-codex"`` → 

50 :class:`calfcord.providers.codex.CodexSubscriptionModelClient` 

51 (routes through ChatGPT Plus/Pro subscription billing rather than 

52 OpenAI API credits; requires ``uv run calfkit-auth codex login``). 

53""" 

54 

55ThinkingEffort = Literal["none", "minimal", "low", "medium", "high", "xhigh", "max"] 

56"""Operator-facing thinking-effort tiers. 

57 

58Seven abstract levels mapped to provider-specific reasoning/thinking 

59parameters in :mod:`calfcord.agents.thinking`. Tier names 

60parallel Claude Code's effort vocabulary; ``minimal`` is the lightest 

61non-zero step (a hair above ``none``); ``xhigh`` is a calfkit-specific 

62step between ``high`` and ``max``. 

63"""

66class AgentDefinition(BaseModel): 

67 """One agent's declarative definition: identity, runtime hints, and system prompt. 

68 

69 Validators mirror the constraints Discord imposes on slash commands and 

70 webhook usernames so misconfiguration fails at load time rather than at 

71 first invocation. The Discord slash command name is always derived as 

72 ``/<agent_id>`` — the ``agent_id`` validator already enforces the 

73 Discord-compatible ``[a-z0-9_-]{1,32}`` shape. 

74 """ 

75 

76 # ``extra="forbid"`` surfaces frontmatter typos (``provder: openai``, 

77 # ``thiking_effort: high``) at parse time rather than silently dropping 

78 # them — important now that a slash command can rewrite the same file. 

79 # It also catches stale ``slash:`` lines left behind from the removal 

80 # of the dedicated slash field. 

81 model_config = ConfigDict(frozen=True, populate_by_name=True, extra="forbid") 

82 

83 agent_id: str = Field(..., alias="name") 

84 description: str 

85 provider: Provider | None = None 

86 model: str | None = None 

87 tools: tuple[str, ...] | None = None 

88 """Tools available to this agent's LLM, resolved against 

89 :data:`calfcord.tools.TOOL_REGISTRY`. 

90 

91 Semantics: 

92 - ``tools:`` omitted from frontmatter (default ``None``) — agent 

93 gets every registered builtin tool. Convenient default for 

94 general-purpose assistant agents. 

95 - ``tools: []`` in frontmatter — agent gets NO tools (text-only). 

96 Explicit opt-out for read-only or routing-style agents. 

97 - ``tools: [a, b]`` — agent gets exactly those tools. 

98 

99 Security note: the "all by default" behaviour means a new 

100 ``agents/<name>.md`` ships with ``terminal`` / ``execute_code`` / 

101 ``write_file`` / ``patch`` access to the workspace bind-mounted into 

102 the ``calfkit-tools`` container unless the operator narrows the list 

103 explicitly. If you need a restricted-tools agent, add the 

104 ``tools:`` line. See :doc:`docs/authoring-agents` for the security 

105 model.""" 

106 thinking_effort: ThinkingEffort | None = None 

107 memory: bool = False 

108 """Opt in to a persistent per-agent notepad. When ``True``, the factory 

109 registers a runtime instructions hook 

110 (:func:`calfcord.agents.memory.memory_instructions`) that 

111 appends the memory-explanation block to this agent's instructions on every 

112 invocation — the stored ``system_prompt`` is left unchanged. The block text 

113 is read only by the bridge (from the editable ``memory_prompt.md``, 

114 overridable via ``CALFCORD_MEMORY_PROMPT_PATH``) and shipped to the agent in 

115 ``deps``; the hook localizes it to ``memory/<agent_id>/`` in the shared 

116 workspace, where the agent keeps one-fact-per-file memories plus a 

117 ``MEMORY.md`` index, managed with the ordinary filesystem tools. 

118 

119 Default ``False`` so existing agents are unchanged. A memory-enabled agent 

120 must have the ``read_file`` and ``write_file`` tools — the factory enforces 

121 this at build time. See :doc:`docs/authoring-agents`.""" 

122 a2a: bool | tuple[str, ...] = True 

123 """Native agent-to-agent messaging (calfkit's auto-injected ``message_agent`` 

124 tool + ``peers=[Messaging(...)]``). 

125 

126 - ``True`` (default) — reach any agent (``Messaging(discover=True)``); the 

127 live peer directory is rendered into the agent's prompt. 

128 - ``False`` — no A2A (the agent gets no ``message_agent`` tool). 

129 - ``[name, ...]`` — restrict to those peers (``Messaging(*names)``). 

130 

131 Replaces the old opt-in (listing the now-removed ``private_chat`` tool); 

132 default-on makes native A2A frictionless.""" 

133 handoff: bool | tuple[str, ...] = True 

134 """Native handoff — transfer the caller's turn to a peer 

135 (``peers=[Handoff(...)]``; the model emits ``HandoffRequest``). 

136 

137 - ``True`` (default) — hand off to any agent (``Handoff(discover=True)``). 

138 - ``False`` — no handoff. 

139 - ``[name, ...]`` — restrict to those targets (``Handoff(*names)``). 

140 

141 Replaces the in-channel ``@<agent_id>`` handoff convention (C7).""" 

142 system_prompt: str 

143 source_path: Path | None = Field(default=None, exclude=True, repr=False) 

144 """Path to the ``.md`` file this definition was parsed from. Set by 

145 :func:`parse_agent_md`; ``None`` for in-memory test constructions. 

146 ``exclude=True`` prevents accidental round-trip into a YAML dump; 

147 ``repr=False`` keeps logs tidy. Required for the ``/thinking-effort`` 

148 slash command's rewrite.""" 

149 

150 @field_validator("agent_id") 

151 @classmethod 

152 def _validate_agent_id(cls, v: str) -> str: 

153 if not AGENT_ID_PATTERN.fullmatch(v): 

154 raise ValueError(f"name must match [a-z0-9_-]{{1,32}}, got {v!r}") 

155 return v 

156 

157 @field_validator("description") 

158 @classmethod 

159 def _validate_description(cls, v: str) -> str: 

160 if not (1 <= len(v) <= 100): 

161 raise ValueError(f"description must be 1-100 chars (Discord slash limit), got {len(v)}") 

162 return v 

163 

164 @field_validator("system_prompt") 

165 @classmethod 

166 def _validate_system_prompt(cls, v: str) -> str: 

167 if not v.strip(): 

168 raise ValueError("system_prompt (markdown body) must be non-empty") 

169 return v

171 @field_validator("a2a", "handoff") 

172 @classmethod 

173 def _normalize_empty_peer_list(cls, v: bool | tuple[str, ...]) -> bool | tuple[str, ...]: 

174 """Treat an empty peer list (``a2a: []`` / ``handoff: []``) as ``False``. 

175 

176 An empty tuple has no useful meaning — the ``message_agent`` / handoff 

177 capability injected with zero reachable peers — and if it reached calfkit 

178 as a bare ``Messaging()`` / ``Handoff()`` it would be rejected (calfkit 

179 needs at least one peer name or ``discover=True``), crashing the agent 

180 worker at boot with an error that names neither the agent nor the field. 

181 Canonicalizing ``()`` → ``False`` here keeps the field a clean tri-state 

182 (``True`` / ``False`` / non-empty tuple) and the persisted/echoed value 

183 canonical. :meth:`AgentFactory._build_peers` independently reads the field 

184 with a truthiness guard, so the two are defense-in-depth: a hand-built 

185 definition that skips validation (``model_construct``) still can't produce 

186 a peerless handle. 

187 """ 

188 if isinstance(v, tuple) and not v: 

189 return False 

190 return v 

191 

192 @field_validator("tools") 

193 @classmethod 

194 def _validate_tools(cls, v: tuple[str, ...] | None) -> tuple[str, ...] | None: 

195 """Syntax-check ``mcp/...`` tool entries; let bare builtin names through. 

196 

197 An ``mcp/...`` entry must be a well-formed selector 

198 (``mcp/<server>`` or ``mcp/<server>/<tool>``); all malformed entries 

199 are aggregated into ONE :class:`ValueError` naming each offending 

200 string, so an operator fixes the whole ``tools:`` line in a single 

201 pass. Whether the named server is configured or running is a 

202 *runtime* concern (the capability view resolves selectors per turn), 

203 deliberately not checked here — this is the authoritative syntactic 

204 gate for every read path: the agent process parses each ``.md`` 

205 through here before booting. 

206 

207 Bare names (anything not starting with ``mcp/`` — ``terminal``, 

208 ``calendar``, …) pass through **untouched**. Whether a bare name 

209 actually resolves to a registered builtin is checked later, by 

210 :meth:`AgentFactory._resolve_tools` against the factory's tool registry 

211 (defaults to :data:`~calfcord.tools.TOOL_REGISTRY`); rejecting 

212 arbitrary bare names here would couple this leaf schema to the tool 

213 registry and break code-built definitions that name tools the local 

214 registry doesn't (yet) hold. 

215 """ 

216 if v is None: 

217 return v 

218 bad: list[str] = [] 

219 for entry in v: 

220 if not is_mcp_selector(entry): 

221 continue 

222 try: 

223 validate_mcp_selector(entry) 

224 except ValueError as exc: 

225 bad.append(f"{entry!r}: {exc}") 

226 if bad: 

227 raise ValueError("malformed MCP tool selector(s) in tools: " + "; ".join(bad)) 

228 return v

231def parse_agent_md(path: Path) -> AgentDefinition: 

232 """Parse one ``agents/<name>.md`` file into an :class:`AgentDefinition`. 

233 

234 Enforces ``path.stem == frontmatter["name"]`` so the on-disk identifier 

235 and the declared name cannot drift. 

236 

237 Raises: 

238 FileNotFoundError: if ``path`` does not exist. 

239 ValueError: if the file has no frontmatter, the YAML is malformed, 

240 the name does not match the filename stem, or any field fails 

241 validation. 

242 """ 

243 try: 

244 post = frontmatter.load(path) 

245 except yaml.YAMLError as e: 

246 # ``frontmatter.load`` lets ``yaml.YAMLError`` propagate unchanged, 

247 # which would escape callers that catch only ``ValueError``. Re-raise 

248 # as ``ValueError`` so the docstring's contract holds and the 

249 # malformed-YAML path is indistinguishable from a malformed-name 

250 # path at the caller's seam. 

251 raise ValueError(f"{path}: malformed YAML frontmatter: {e}") from e 

252 metadata = dict(post.metadata) 

253 body = post.content 

254 

255 if not metadata: 

256 raise ValueError(f"{path}: missing YAML frontmatter") 

257 

258 declared_name = metadata.get("name") 

259 expected_name = path.stem 

260 if declared_name != expected_name: 

261 raise ValueError( 

262 f"{path}: frontmatter name={declared_name!r} does not match filename stem {expected_name!r}" 

263 ) 

264 

265 metadata["system_prompt"] = body.strip() 

266 # Resolve so the path remains valid even if the process later 

267 # ``os.chdir``s — the bridge daemon doesn't today, but a future 

268 # plugin or signal handler could. 

269 metadata["source_path"] = path.resolve() 

270 return AgentDefinition(**metadata)
