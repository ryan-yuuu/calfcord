1"""``disco agent create [<name>]`` — the reusable agent-creation flow. 

2 

3This is the one place the "name → describe → provider/model → tools → write" 

4sequence lives, so the two surfaces that need it — the standalone ``agent 

5create`` command *and* ``init``'s first-run setup — can never drift on 

6*how* an agent is brought into being. :func:`create_agent` is the extracted 

7flow; :func:`run` is the thin ``agent create`` wrapper around it (no seed prune, 

8offers the optional ``$EDITOR`` prompt step, prints the restart guidance). 

9 

10Two design rules keep the two callers honest: 

11 

12* **``create_agent`` never touches ``CALFKIT_AGENT_DEFAULT_PROVIDER``.** The 

13 agent it writes carries an *explicit* ``provider``/``model`` in its 

14 frontmatter, so the install-wide default-provider env var is irrelevant to it 

15 — that env default is purely ``init``'s concern (first-run wants a sensible 

16 default for *future* agents). Writing it here would let ``agent create`` of a 

17 one-off OpenAI agent silently flip the install default, surprising the next 

18 ``init`` re-run. 

19 

20* **Provider/model/tools all flow through the validated seams.** 

21 :func:`~calfcord.cli._providers.configure_provider` owns provider-select, 

22 credential capture, and the live model pick (so an operator can never type a 

23 slug the provider rejects); :func:`~calfcord.cli._agents.pick_tools` owns the 

24 pre-checked tool checkbox; :func:`~calfcord.cli._agents.write_agent` owns the 

25 validate-before-write disk path. This module only sequences them. 

26 

27``configure_provider`` is imported at module scope (not lazily) so tests can 

28monkeypatch ``agent_create.configure_provider`` to a fixed ``(provider, model)`` 

29and drive the whole flow without a provider SDK, network, key, or OAuth — the 

30same pattern the ``init`` tests use. 

31""" 

32

56# never writes it — see the module docstring's "never touches" rule. 

57_DEFAULT_PROVIDER_VAR = "CALFKIT_AGENT_DEFAULT_PROVIDER"

60class CreatedAgent(NamedTuple): 

61 """What :func:`create_agent` produced: the agent's resolved ``name`` and ``provider``. 

62 

63 A named pair (not a bare ``tuple[str, str]``) so the two same-typed strings 

64 can't be unpacked in the wrong order — callers read ``.name`` / ``.provider``. 

65 ``init`` uses ``.provider`` to persist the install default; ``agent create`` 

66 uses ``.name`` for its restart hint. 

67 """ 

68 

69 name: str 

70 provider: str 

71

73def create_agent( 

74 prompter: Prompter, 

75 *, 

76 agents_dir: Path, 

77 env_path: Path, 

78 name_default: str | None = None, 

79 prune_seed: bool = False, 

80 offer_prompt: bool = True, 

81) -> CreatedAgent: 

82 """Run the shared create flow and return the created agent's ``name`` + ``provider``. 

83 

84 The single create sequence both ``agent create`` (``prune_seed=False``, 

85 ``offer_prompt=True``) and ``init``'s first-run setup (``prune_seed=True``, 

86 ``offer_prompt=False``) build on, so the two can't drift on how an agent is 

87 created. Steps: 

88 

89 1. **Name.** Defaults to ``name_default`` when given, else the lone existing 

90 agent's stem (a re-run editing it in place), else the starter name. The 

91 typed value is slugified so it can't yield an invalid filename; a blank 

92 answer keeps the default. 

93 2. **Description.** Pre-filled from the target agent's current description on 

94 a re-run (so editing in place shows the existing value), else the seed 

95 default; a blank answer falls back to the seed default. 

96 3. **Provider + credentials + model.** Delegated wholesale to 

97 :func:`~calfcord.cli._providers.configure_provider`, which owns 

98 provider-select, key/Codex auth, and the live model pick. The provider 

99 menu is pre-selected from the install default-provider env var (read, 

100 never written here) so a fresh agent biases toward the operator's usual 

101 choice; ``current_model`` pre-selects an existing agent's model on a 

102 re-run. 

103 4. **Tools.** The pre-checked builtin-tool checkbox via 

104 :func:`~calfcord.cli._agents.pick_tools`. 

105 5. **Write.** :func:`~calfcord.cli._agents.write_agent` (validate before 

106 write; ``prune_seed`` only when the caller opted in). 

107 6. **Optional prompt edit.** When ``offer_prompt`` and the operator 

108 confirms, open the new agent's system prompt in ``$EDITOR`` via 

109 :func:`calfcord.cli.agent_edit.edit_system_prompt` (imported lazily to 

110 keep this module free of the subprocess/editor concern unless used). 

111 

112 Returns the created agent's ``name`` and ``provider`` (as a 

113 :class:`CreatedAgent`) so the caller can word its own success/next-steps 

114 guidance (``init`` persists the provider as the install default; ``agent 

115 create`` just names the agent in its restart hint). Lets :class:`ValueError` / 

116 :class:`OSError` from :func:`~calfcord.cli._agents.write_agent` propagate — the 

117 caller decides how to report a write failure (and must not print a success 

118 banner on one). 

119 """ 

120 current = read_env(env_path) 

121 

122 # 1. Name. ``name_default`` (explicit) wins; otherwise default to the lone 

123 # existing agent (re-run editing it) or the starter on a fresh install. 

124 if name_default is None: 

125 existing = detect_agents(agents_dir) 

126 name_default = existing[0] if len(existing) == 1 else STARTER_AGENT_NAME 

127 typed_name = prompter.text("Agent name:", default=name_default) 

128 name = slug_stem(typed_name) if typed_name.strip() else name_default 

129 

130 # 2. Description. Pre-fill from the target (if it already exists) so a re-run 

131 # shows the current value; a blank answer falls back to the seed default. 

132 prior = existing_agent(agents_dir, name) 

133 desc_default = (prior.description if prior else None) or DEFAULT_DESCRIPTION 

134 typed_desc = prompter.text("Agent description:", default=desc_default) 

135 description = typed_desc.strip() or DEFAULT_DESCRIPTION 

136 

137 # 3. Provider + credentials + model. ``configure_provider`` writes only the 

138 # credential side effect; we read (never write) the install default-provider 

139 # env var purely to pre-select the menu. 

140 provider, model = configure_provider( 

141 prompter, 

142 env_path=env_path, 

143 current=current, 

144 default_provider=current.get(_DEFAULT_PROVIDER_VAR) or "anthropic", 

145 cheap=False, 

146 current_model=prior.model if prior else None, 

147 ) 

148 

149 # 4. Tools. 

150 tools = pick_tools(prompter, name) 

151 

152 # 5. Write (validate-before-write; prune only when the caller opted in). 

153 md_path = write_agent( 

154 agents_dir, 

155 name=name, 

156 description=description, 

157 provider=provider, 

158 model=model, 

159 tools=tools, 

160 prune_seed=prune_seed, 

161 ) 

162 

163 # 6. Optional system-prompt edit. Imported lazily so merely importing this 

164 # module (which ``init`` does at startup) never pulls in the editor/ 

165 # subprocess machinery unless an operator actually opts to edit the prompt. 

166 if offer_prompt and prompter.confirm( 

167 "Edit this agent's system prompt now? (opens $EDITOR)", default=False 

168 ): 

169 from calfcord.cli.agent_edit import edit_system_prompt 

170 

171 edit_system_prompt(md_path) 

172 

173 return CreatedAgent(name=name, provider=provider)

176def run(prompter: Prompter, *, agents_dir: Path, env_path: Path, name: str | None = None) -> int: 

177 """``disco agent create [<name>]``: create one agent and return an exit code. 

178 

179 The standalone create command: it runs :func:`create_agent` with 

180 ``prune_seed=False`` (adding an agent must never delete the operator's 

181 starter — only ``init``'s first-run prunes a *pristine* seed) and 

182 ``offer_prompt=True`` (the operator can jump straight into editing the new 

183 agent's system prompt). A given ``name`` pre-fills the name prompt; the 

184 operator can still rename at the prompt. 

185 

186 On success it names the created agent then prints the terse next-step block 

187 (behavior #3): a sentence ending in a colon, a blank line, the indented 

188 command. A brand-new agent comes online via the roster verb (the new 

189 substrate/roster model), so the steer is ``disco agent start <name>`` — not 

190 the old runner-restart banner. Per the CLI error-handling convention, a write 

191 failure (``ValueError``/``OSError`` from the validate-before-write path) is 

192 reported as a single ``error:`` line and returns 1 with no success banner — 

193 printing "Created agent ..." on a failed write would send the operator off to 

194 boot processes against an agent that isn't there. 

195 """ 

196 try: 

197 created = create_agent( 

198 prompter, 

199 agents_dir=agents_dir, 

200 env_path=env_path, 

201 name_default=name, 

202 prune_seed=False, 

203 offer_prompt=True, 

204 ) 

205 except (ValueError, OSError) as e: 

206 # The create path validates before writing, so this is either an invalid 

207 # value the validator rejected or a filesystem failure during the atomic 

208 # write — both leave no usable agent on disk. Report and stop without a 

209 # success banner. 

210 print(f"error: could not create agent {(name or '?')!r}: {e}") 

211 return 1 

212 

213 print(f"Created agent {created.name!r}.") 

214 print(f"Bring {created.name} online:\n\n disco agent start {created.name}") 

215 return 0
