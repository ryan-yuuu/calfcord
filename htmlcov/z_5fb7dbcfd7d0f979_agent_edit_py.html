1"""``calfcord agent edit [<name>]`` — the interactive field-menu editor. 

2 

3The workhorse of the editing surface: resolve an agent, then loop a menu 

4of its editable fields (sourced from :data:`calfcord.cli._fields.FIELDS`, the one 

5registry every edit/set/show surface shares) where each row shows the field's 

6current value and selecting it edits exactly that field. This module also owns 

7:func:`edit_system_prompt`, the ``$EDITOR`` helper the menu's *prompt* row and 

8``agent create``'s optional prompt step both reuse. 

9 

10Three design rules shape the loop: 

11 

12* **Re-read before every edit.** Each iteration re-parses the ``.md`` with 

13 :func:`~calfcord.agents.definition.parse_agent_md` so the menu always shows the 

14 freshly-written value (a just-edited field reflects immediately) and so an edit 

15 validates against current on-disk state, never a stale snapshot. 

16 

17* **A bad value must not crash the menu.** Every field edit is wrapped so a 

18 validation/OS error (an out-of-range ``history_turns``, a read-only file) 

19 prints one ``error:`` line and *continues* the loop. The writes go through the 

20 validate-before-write seams (:func:`calfcord.cli._fields.write_simple_field`, 

21 :func:`calfcord.agents.md_writer`), so a rejected value leaves the on-disk file 

22 untouched — the operator can simply retry. 

23 

24* **Each field reaches its own validated seam.** Simple scalars ride 

25 :func:`~calfcord.cli._fields.write_simple_field`; the provider+model pair rides 

26 the shared provider flow then :func:`calfcord.agents.md_writer._update_fields`; 

27 ``tools`` delegates to the existing :func:`calfcord.cli.agent_tools.run` 

28 checkbox editor; ``prompt`` rides :func:`edit_system_prompt`. The menu never 

29 re-encodes a field's validation — it routes to the seam that owns it. 

30 

31Rename and delete are deliberately *not* in this menu — they change the agent's 

32identity/existence (and its filename), so they are separate commands. 

33"""

58def edit_system_prompt(md_path: Path) -> None: 

59 """Open the agent's system-prompt **body** in ``$EDITOR`` and save the result. 

60 

61 The system prompt is free-form prose, so an inline single-line prompt is the 

62 wrong tool — we hand the operator their real editor. The current body is 

63 written to a temp ``.md`` file, ``$EDITOR`` (or ``vi`` when unset) is launched 

64 on it, and the edited contents are read back and persisted via the 

65 validate-before-write :func:`calfcord.agents.md_writer.update_system_prompt`. 

66 

67 Defensive on every failure mode that an interactive editor invites, because 

68 this runs inside the edit menu and must never let an exception escape and 

69 tear the menu down: 

70 

71 * **No change / emptied.** If the operator saves without changes, or empties 

72 the file (a whitespace-only body the validator would reject anyway), print 

73 a note and return without writing — leaving the existing prompt intact. 

74 * **Editor can't be launched.** A missing ``$EDITOR`` binary 

75 (:class:`FileNotFoundError`) prints a clear hint to set ``$EDITOR`` and 

76 returns, rather than surfacing a raw stack trace. 

77 * **Validation / OS error.** A rejected body or a filesystem error during the 

78 atomic write prints one ``error:`` line and returns; the on-disk file is 

79 left untouched by the validate-before-write seam. 

80 

81 Never raises out of the menu — every recoverable failure is reported and 

82 swallowed here. 

83 """ 

84 try: 

85 current_body = parse_agent_md(md_path).system_prompt 

86 except (ValueError, OSError) as e: 

87 # An unreadable/malformed .md can't be edited via $EDITOR meaningfully; 

88 # report and return so the menu (or create flow) keeps going. 

89 print(f"error: cannot read agent prompt: {e}") 

90 return 

91 

92 # A temp .md so an editor with Markdown niceties keys off the extension; in 

93 # the same dir would be ideal for atomicity but the write goes through the 

94 # md_writer's own atomic path, so a plain system temp file is fine here. 

95 fd, tmp_name = tempfile.mkstemp(prefix="calfcord-prompt-", suffix=".md") 

96 tmp_path = Path(tmp_name) 

97 try: 

98 with os.fdopen(fd, "w", encoding="utf-8") as handle: 

99 handle.write(current_body) 

100 

101 # ``shlex.split`` so an EDITOR carrying args ("code --wait", "emacs -nw") 

102 # is honoured rather than treated as one impossible binary name. 

103 editor = os.environ.get("EDITOR") or "vi" 

104 argv = [*shlex.split(editor), str(tmp_path)] 

105 try: 

106 subprocess.run(argv, check=True) 

107 except FileNotFoundError: 

108 # The configured editor binary doesn't exist on PATH — the single 

109 # failure that a generic "error:" line would leave the operator 

110 # unable to act on, so hint at the concrete fix. 

111 print( 

112 f"error: could not launch editor {editor!r}; set $EDITOR to an " 

113 f"installed editor (e.g. EDITOR=nano) and try again." 

114 ) 

115 return 

116 except (subprocess.CalledProcessError, OSError) as e: 

117 # The editor exited non-zero / couldn't run — treat as "operator 

118 # aborted", don't write, but say why. 

119 print(f"error: editor exited without saving ({e}); prompt unchanged.") 

120 return 

121 

122 try: 

123 new_body = tmp_path.read_text(encoding="utf-8") 

124 except (ValueError, OSError) as e: 

125 # Reading the edited temp file back can fail — e.g. the operator saved 

126 # a non-UTF-8 body (``UnicodeDecodeError``, a ``ValueError``). Catch it 

127 # here so this function honours its "never raises out" contract even 

128 # when called from ``agent create``'s prompt step, which has no menu 

129 # loop to absorb a stray exception. 

130 print(f"error: could not read the edited prompt ({e}); prompt unchanged.") 

131 return 

132 finally: 

133 tmp_path.unlink(missing_ok=True) 

134 

135 # No-op guard: unchanged or emptied → don't touch the file. An empty body 

136 # would fail the validator anyway; reporting it as "unchanged" reads better 

137 # than an "error:" for the common "saved without editing" case. 

138 if not new_body.strip() or new_body == current_body: 

139 print("Prompt unchanged.") 

140 return 

141 

142 try: 

143 md_writer.update_system_prompt(md_path, new_body) 

144 except (ValueError, OSError) as e: 

145 print(f"error: {e}") 

146 return 

147 print("Updated system prompt.")

150def _resolve_agent(prompter: Prompter, *, agents_dir: Path, name: str | None) -> Path | None: 

151 """Resolve the agent ``.md`` to edit, or ``None`` after printing why not. 

152 

153 Mirrors :func:`calfcord.cli.agent_tools._resolve_agent` so the two editors 

154 agree on resolution: a given ``name`` must exist (an explicit request for a 

155 missing agent is an error, not a fallback to the picker); an omitted ``name`` 

156 lists the detected agents and prompts, with an empty dir an error too. 

157 Returning ``None`` lets :func:`run` map every "can't proceed" case to a 

158 single already-printed message + exit code 1. 

159 """ 

160 if name is not None: 

161 md_path = agents_dir / f"{name}.md" 

162 if not md_path.is_file(): 

163 print(f"error: no agent {name!r} in {agents_dir} (expected {md_path})") 

164 return None 

165 return md_path 

166 

167 agents = detect_agents(agents_dir) 

168 if not agents: 

169 print(f"no agents in {agents_dir}") 

170 return None 

171 chosen = prompter.select( 

172 "Which agent do you want to edit?", 

173 [Choice(a, a) for a in agents], 

174 ) 

175 return agents_dir / f"{chosen}.md"

187 """Edit the one field ``field_key`` and report whether anything changed. 

188 

189 Re-parses the ``.md`` fresh so the prompt's default reflects current on-disk 

190 state, then dispatches by the field's :attr:`~calfcord.cli._fields.Field.kind` 

191 to the seam that owns that field's validation: 

192 

193 * ``text`` / ``select`` / ``int`` / ``bool`` ride the one validated-atomic 

194 :func:`~calfcord.cli._fields.write_simple_field` (which coerces and lets 

195 :class:`~calfcord.agents.definition.AgentDefinition` enforce the 

196 constraint), and only write when the value actually changed; 

197 * ``provider_model`` runs the shared provider flow (so a model the provider 

198 rejects can't be typed) then writes provider+model together via the one 

199 validated-atomic metadata seam; 

200 * ``tools`` delegates to the existing checkbox editor; 

201 * ``prompt`` delegates to :func:`edit_system_prompt`. 

202 

203 Returns ``True`` when an edit was attempted (the menu uses this only to decide 

204 whether to print the restart hint at the end). The simple-scalar branches 

205 return ``False`` on a no-op (the value equalled the current one); the compound 

206 branches (``provider_model`` / ``tools`` / ``prompt``) delegate to sub-editors 

207 that report their own outcome, so they conservatively return ``True`` even 

208 when nothing changed — the cost is at most a redundant restart hint. Raises 

209 :class:`ValueError` / :class:`OSError` on a bad value or a filesystem failure — 

210 :func:`run` catches it, prints one ``error:`` line, and continues the loop (the 

211 validated-write seams leave the file untouched on failure). 

212 """

214 field = FIELDS_BY_KEY[field_key] 

215

221 return True 

222 return False 

223

227 new = prompter.select(field.label, choices, default=current if current else None)

230 return True 

231 return False 

232

237 default=str(current),

241 return True 

242 return False

251 return True

283def run(prompter: Prompter, *, agents_dir: Path, env_path: Path, name: str | None = None) -> int: 

284 """``calfcord agent edit [<name>]``: loop a field menu over one agent. 

285 

286 Resolves the agent (given ``name`` must exist; omitted ``name`` prompts a 

287 picker; an empty agents dir is an error), then loops: show each editable 

288 field with its current value plus a ``✓ Done`` row, edit the chosen field 

289 through its owning validated seam, and re-read before the next iteration so 

290 the menu always reflects disk. Per the CLI error-handling convention, a bad 

291 value or a filesystem hiccup during one edit prints a single ``error:`` line 

292 and the loop continues (the validate-before-write seams leave the file 

293 untouched), so a typo never drops the operator out of the menu. 

294 

295 Returns 1 (with an already-printed message) when no agent can be resolved; 

296 otherwise 0, printing the restart hint when at least one field was changed. 

297 """ 

298 md_path = _resolve_agent(prompter, agents_dir=agents_dir, name=name) 

299 if md_path is None: 

300 return 1 

301 agent_name = md_path.stem 

302 

303 changed = False 

304 while True: 

305 try: 

306 defn = parse_agent_md(md_path) 

307 except (ValueError, OSError) as e: 

308 # A malformed/unreadable .md can't drive the menu — report and exit 

309 # non-zero rather than loop forever on an unparseable file. 

310 print(f"error: cannot read agent {agent_name!r}: {e}") 

311 return 1 

312 

313 rows = [Choice(f.key, f"{f.label} — {render_value(defn, f)}") for f in FIELDS] 

314 rows.append(Choice(_DONE, "✓ Done")) 

315 choice = prompter.select("What do you want to change?", rows) 

316 if choice == _DONE: 

317 break 

318 

319 try: 

320 if _edit_field( 

321 prompter, 

322 md_path=md_path, 

323 agents_dir=agents_dir, 

324 env_path=env_path, 

325 name=agent_name, 

326 field_key=choice, 

327 ): 

328 changed = True 

329 except (ValueError, OSError) as e: 

330 # A rejected value (an out-of-range history_turns, a bad 

331 # thinking_effort choice, an unknown tool) or a filesystem error — the 

332 # validated-write path left the file untouched, so just report and keep 

333 # looping. (The memory→fs-tools requirement is NOT enforced here; it is 

334 # a build-time check in AgentFactory, so enabling memory writes fine and 

335 # surfaces at the next calfkit-agent boot.) 

336 print(f"error: {e}") 

337 

338 if changed: 

339 # The terse next-step block (behavior #3): a sentence ending in a colon, a 

340 # blank line, the two-space-indented command. A config edit takes effect on 

341 # a running agent via the roster `restart` verb (the node bakes its config at 

342 # construction); the parenthetical flags that a provider/key change can 

343 # affect every agent sharing that provider, so a same-provider fleet may need 

344 # restarting too. ``defn`` is the last re-read definition (the menu re-parses 

345 # each iteration), so it reflects the just-applied provider. 

346 print( 

347 f"Restart {agent_name} to apply (and any other agents on {defn.provider} " 

348 f"if the provider/key changed):\n\n calfcord agent restart {agent_name}" 

349 ) 

350 return 0
