1"""``calfcord init`` — one continuous, resumable guided setup that ends LIVE. 

2 

3This is the onboarding alternative to hand-editing ``.env`` *and* hand-writing an 

4``agents/<name>.md``. It walks the operator through one agent end to end, then — 

5on a native install — *opens the workspace, brings the agent online, and waits 

6until it sees the first real reply in Discord*. The flow is the §4.6 / §11 

7"ends-live" experience: time-to-first-reply over everything. 

8 

9Composition, not reinvention 

10---------------------------- 

11``init`` is a **composer**. Each cohesive unit lives in its own module and ``init`` 

12only sequences them, so the wizard stays unit-testable and the pieces stay 

13reusable: 

14 

15* **Agent + provider + model** — delegated wholesale to 

16 :func:`calfcord.cli.agent_create.create_agent` (the ONE shared create flow, 

17 which ``agent create`` also uses, so the two can't drift). ``init`` opts into 

18 pruning the pristine starter and persists the chosen provider as the install 

19 default. 

20* **Discord** — :func:`_run_discord` composes :mod:`calfcord.cli.discord_discovery` 

21 (verify-token-on-paste, the invite link + intents reminder, block-and-poll 

22 until the bot joins, then guild / *postable*-channel pick-lists) in place of 

23 the old "paste a numeric ID" prompts (§4.5). 

24* **Live finish** — :func:`_run_finish` composes 

25 :func:`calfcord.supervisor.lifecycle.start` (substrate, health-gated) → 

26 :func:`calfcord.supervisor.roster.agent_start` (the agent clocks in) → an 

27 in-flow ``@<agent> hello`` prompt → :func:`first-reply detection 

28 <calfcord.control_plane.first_reply.wait_for_first_reply>` (§4.6 / §12.6). On a 

29 dev run (no install) or a missing supervisor binary it DEGRADES to honest 

30 manual next-steps rather than orchestrating something it cannot. 

31* **Resumability** — :mod:`calfcord.cli.setup_state` records *which steps are 

32 done* so a crash / Ctrl-C / the unavoidable browser detour resumes ("Welcome 

33 back …") instead of restarting. The checkpoint is **advisory** (§12.7): every 

34 resumed step RE-VERIFIES the real artifact (agent ``.md`` parses? token still 

35 valid?) before skipping — the world is ground truth, the checkpoint only 

36 chooses *where* to resume. 

37 

38Injected seams 

39-------------- 

40All prompting goes through an injected :class:`Prompter`; every world-touching 

41dependency — the Discord HTTP calls, the substrate/roster coroutines, the 

42first-reply watcher, the process-compose binary probe, and the clock — is a 

43keyword-only injectable defaulting to the real thing. So the whole wizard runs in 

44a unit test with no TTY, no Discord, no broker, and no supervisor. 

45 

46Two invariants the design pins: 

47 

48* **Idempotent and non-destructive to secrets.** Re-running treats an empty 

49 answer as "keep what's there" for every ``.env`` secret, and defaults a re-run 

50 to the saved (working) guild/channel binding rather than clobbering it (§12.7). 

51* **No green light that lies.** The finish only celebrates on a *detected* reply; 

52 a clean timeout downgrades to an honest "try it yourself / run doctor" hint, 

53 and a substrate that never reaches ready stops the flow instead of clocking an 

54 agent into a workspace that isn't up (§12.6). 

55"""

91def resolve_paths(home: Path | None) -> tuple[Path, Path]: 

92 """Resolve ``(env_path, agents_dir)`` for the current run. 

93 

94 Native installs pass ``home`` (``$CALFCORD_HOME``): config lives at 

95 ``home/config/.env`` and agents at ``home/agents`` — unless the operator 

96 pinned a different agents dir via ``CALFKIT_AGENTS_DIR``, which the shim and 

97 runners already honour, so ``init``'s detection must agree with them. 

98 

99 Dev / ``uv run calfcord-cli init`` passes ``home=None``: config is the 

100 project-local ``./.env`` and agents the project-local ``./agents`` (again 

101 overridable by ``CALFKIT_AGENTS_DIR``), matching the non-shim defaults. 

102 """ 

103 agents_override = os.environ.get("CALFKIT_AGENTS_DIR") 

104 if home is not None: 

105 env_path = home / "config" / ".env" 

106 agents_dir = Path(agents_override) if agents_override else home / "agents" 

107 else: 

108 env_path = Path(".env") 

109 agents_dir = Path(agents_override) if agents_override else Path("agents") 

110 return env_path, agents_dir

135 return False 

136 return True

141 *, 

142 env_path: Path, 

143 agents_dir: Path, 

144 home: Path | None = None, 

145 server_urls: str = "localhost", 

146 # --- injected world-touching seams (default to the real thing) ---------- 

147 verify_identity_fn: Callable[..., BotIdentity] | None = None, 

148 poll_joined_fn: Callable[..., list[Guild]] | None = None, 

149 list_guilds_fn: Callable[..., list[Guild]] | None = None, 

150 list_channels_fn: Callable[..., ChannelListing] | None = None, 

151 start_fn: Callable[..., Awaitable[int]] | None = None,

154 pc_binary_fn: Callable[[], str] | None = None,

157 """Run the guided, resumable, ends-live setup flow and return an exit code. 

158 

159 Phases, in order: **(1)** agent identity + provider + model + tools + write 

160 (the shared :func:`create_agent`), **(2)** Discord (:func:`_run_discord`), 

161 **(3)** broker, **(4)** the live finish (:func:`_run_finish`). A checkpoint is 

162 saved after each completed phase so a Ctrl-C resumes; each resumed phase 

163 re-verifies its real artifact before skipping (advisory, §12.7). 

164 

165 Every ``.env`` write goes through :func:`_envfile.upsert`; an empty secret 

166 answer keeps the existing value (re-run safe). ``server_urls`` is the broker 

167 URL ``main.py`` sampled from ``CALF_HOST_URL`` BEFORE the wizard ran; it is a 

168 pre-wizard hint only — the broker phase (§3) may write a different 

169 ``CALF_HOST_URL``, so the live finish re-reads the EFFECTIVE value from the 

170 just-written ``.env`` (same ``value or "localhost"`` default the runners use) 

171 rather than trusting this. The injected seams are the test surface — 

172 production defaults wire the real :mod:`discord_discovery`, :mod:`supervisor`, 

173 and first-reply modules. 

174 """ 

175 verify_identity_fn = verify_identity_fn or discord_discovery.verify_bot_identity 

176 poll_joined_fn = poll_joined_fn or discord_discovery.poll_until_joined 

177 list_guilds_fn = list_guilds_fn or discord_discovery.list_guilds 

178 list_channels_fn = list_channels_fn or discord_discovery.list_postable_channels 

179 

180 checkpoint_file = setup_state.checkpoint_path(home) 

181 checkpoint = setup_state.load(checkpoint_file) or setup_state.SetupCheckpoint() 

182 

183 print("calfcord init — configuring", env_path) 

184 # Advisory resume greeting: only when the checkpoint claims the agent step is 

185 # done AND the real .md still parses (re-verify, never trust the flag alone). 

186 resuming = ( 

187 checkpoint.provider_done 

188 and checkpoint.agent_name is not None 

189 and _agent_md_parses(agents_dir, checkpoint.agent_name) 

190 ) 

191 if resuming: 

192 # Honest wording (nit #18a): the resume RE-WALKS the create flow rather 

193 # than skipping it, so don't claim the agent step is settled. We pre-fill 

194 # the create defaults from the saved agent (a blank answer keeps it) — the 

195 # re-walk confirms/edits in place, it doesn't restart from scratch. 

196 print( 

197 f"Welcome back — picking up where you left off (agent {checkpoint.agent_name}). " 

198 "Press enter to keep each saved answer." 

199 ) 

200 print() 

201 

202 # --- Phase 1: agent identity + provider + model + tools + write -------- 

203 # Delegated wholesale to the shared create flow so ``agent create`` and 

204 # ``init`` can't drift on how an agent is made. A write failure means no 

205 # usable agent landed, so abort before Discord / broker / the live finish. 

206 try: 

207 created = create_agent( 

208 prompter, 

209 agents_dir=agents_dir, 

210 env_path=env_path, 

211 # On resume, default the name prompt to the agent the operator was 

212 # mid-creating (nit #18b): in a 2+-agent install ``create_agent``'s own 

213 # default falls back to the starter (its lone-existing rule needs 

214 # exactly one agent), so without this the re-walk would re-create under 

215 # the wrong name. A fresh run passes None and lets that default logic own it. 

216 name_default=checkpoint.agent_name if resuming else None, 

217 prune_seed=True, 

218 offer_prompt=False, 

219 ) 

220 except (ValueError, OSError) as e: 

221 print(f"error: could not create agent: {e}") 

222 return 1 

223 name = created.name 

224 _envfile.upsert(env_path, {_DEFAULT_PROVIDER_VAR: created.provider}) 

225 checkpoint = checkpoint.model_copy(update={"provider_done": True, "agent_name": name}) 

226 setup_state.save(checkpoint_file, checkpoint, now=now) 

227 print() 

228 

229 # --- Phase 2: Discord (verify → invite → poll → pick guild/channel) ---- 

230 checkpoint = _run_discord( 

231 prompter, 

232 env_path=env_path, 

233 checkpoint=checkpoint, 

234 verify_identity_fn=verify_identity_fn, 

235 poll_joined_fn=poll_joined_fn, 

236 list_guilds_fn=list_guilds_fn, 

237 list_channels_fn=list_channels_fn, 

238 ) 

239 setup_state.save(checkpoint_file, checkpoint, now=now) 

240 print() 

241 

242 # --- Phase 3: broker --------------------------------------------------- 

243 _run_broker(prompter, env_path=env_path) 

244 checkpoint = checkpoint.model_copy(update={"broker_done": True}) 

245 setup_state.save(checkpoint_file, checkpoint, now=now) 

246 print() 

247 

248 # Re-read the EFFECTIVE broker URL the broker phase just wrote, rather than 

249 # the pre-wizard ``server_urls`` (sampled by main.py BEFORE the wizard ran). 

250 # The operator can configure a different broker inside the wizard, so the 

251 # live finish's lifecycle.start broker probe AND the first-reply watcher must 

252 # connect to what is now on disk — using the SAME ``value or "localhost"`` 

253 # default the runners (main.py ``_run_lifecycle``) resolve from the env, so 

254 # all three agree on the broker the install actually talks to. 

255 effective_server_urls = _envfile.read_env(env_path).get(_BROKER_VAR) or "localhost" 

256 

257 # --- Phase 4: live finish (or honest degrade) -------------------------- 

258 return _run_finish( 

259 prompter, 

260 name=name, 

261 home=home, 

262 agents_dir=agents_dir, 

263 env_path=env_path, 

264 server_urls=effective_server_urls, 

265 start_fn=start_fn, 

266 agent_start_fn=agent_start_fn, 

267 first_reply_fn=first_reply_fn, 

268 pc_binary_fn=pc_binary_fn, 

269 )

272def _run_discord( 

273 prompter: Prompter, 

274 *, 

275 env_path: Path, 

276 checkpoint: setup_state.SetupCheckpoint, 

277 verify_identity_fn: Callable[..., BotIdentity], 

278 poll_joined_fn: Callable[..., list[Guild]], 

279 list_guilds_fn: Callable[..., list[Guild]], 

280 list_channels_fn: Callable[..., ChannelListing], 

281) -> setup_state.SetupCheckpoint: 

282 """The Discord sub-flow: validate-on-paste → invite → poll → pick (§4.5/§12.6). 

283 

284 Composes :mod:`discord_discovery` to replace the old numeric-ID prompts. The 

285 bot token is captured (keep-existing-on-empty), verified the instant it is 

286 pasted (echoing the bot's identity), then the operator invites the bot, the 

287 wizard block-and-polls until it joins, and guild / *postable*-channel 

288 pick-lists persist ``DISCORD_GUILD_ID`` / ``DISCORD_DEFAULT_CHANNEL_ID``. 

289 

290 Returns the checkpoint advanced with ``discord_done`` and the chosen 

291 non-secret guild/channel IDs. The whole sub-flow is non-fatal: a join timeout 

292 or zero postable channels surfaces the §12.6 actionable hint and returns what 

293 progress it could make rather than aborting the wizard. 

294 """ 

295 current = _envfile.read_env(env_path) 

296 print("Discord setup (the wizard discovers your server and channel — no IDs to paste).") 

297 

298 token = _capture_token(prompter, env_path, current, verify_identity_fn) 

299 if not token: 

300 # No token at all (fresh run, skipped): nothing to discover. The bridge 

301 # will fail-fast later; we surface it but keep going so the rest of the 

302 # config still lands and a re-run can finish Discord. 

303 print(" no Discord token set — skipping discovery; re-run init to finish Discord.") 

304 return checkpoint 

305 

306 # Invite step: print the ready-made link + the privileged-intents reminder + 

307 # the resumability banner BEFORE the wait (§12.6 — Ctrl-C is safe here). 

308 app_id = _capture_app_id(prompter, env_path, current) 

309 print() 

310 print("Invite the bot to your server:") 

311 print(f" {discord_discovery.invite_url(app_id)}") 

312 print(f" {discord_discovery.INTENTS_REMINDER}") 

313 print(" (Ctrl-C is safe & resumable — re-run `calfcord init` to pick up where you left off.)") 

314 print() 

315 print("Waiting for the bot to join a server…") 

316 

317 try: 

318 guilds = poll_joined_fn(token) 

319 except discord_discovery.DiscordJoinTimeoutError: 

320 # The user never authorized within the budget. Surface the common causes 

321 # + the "I don't have a server" branch (§12.6) and degrade — the binding 

322 # is unset, but the rest of init still completes and a re-run finishes it. 

323 print(" the bot did not join a server in time. Common causes:") 

324 print(" - did you click Authorize on the invite link?") 

325 print(" - are the Message Content and Server Members intents enabled?") 

326 print(" - do you have Manage Server on the server you tried to add it to?") 

327 print(" No server yet? Create one in Discord (the + button), then re-run `calfcord init`.") 

328 return checkpoint 

329 except discord_discovery.DiscordDiscoveryError as e: 

330 # Rate-limited / unreachable on the one-shot poll: surface and degrade. 

331 print(f" could not confirm the bot joined ({e}); re-run init to finish Discord.") 

332 return checkpoint 

333 

334 guild_id = _pick_guild(prompter, guilds, default=checkpoint.guild_id) 

335 if guild_id is None: 

336 return checkpoint 

337 _envfile.upsert(env_path, {"DISCORD_GUILD_ID": guild_id}) 

338 

339 channel_id = _pick_channel( 

340 prompter, list_channels_fn, token, guild_id, default=checkpoint.channel_id 

341 ) 

342 if channel_id is None: 

343 # Guild bound but no postable channel chosen (zero postable / surfaced 

344 # gap). Record the guild progress; the channel can be picked on a re-run. 

345 return checkpoint.model_copy(update={"guild_id": guild_id}) 

346 _envfile.upsert(env_path, {"DISCORD_DEFAULT_CHANNEL_ID": channel_id}) 

347 

348 return checkpoint.model_copy( 

349 update={"discord_done": True, "guild_id": guild_id, "channel_id": channel_id} 

350 )

353def _capture_token( 

354 prompter: Prompter, 

355 env_path: Path, 

356 current: dict[str, str], 

357 verify_identity_fn: Callable[..., BotIdentity], 

358) -> str: 

359 """Prompt for the bot token (keep-existing-on-empty) and verify it on paste. 

360 

361 Returns the token in effect (the freshly pasted one, or the kept existing 

362 one). A rejected token is fatal for *that* value, so we re-prompt for a fresh 

363 one (re-prompting the same token would be pointless, §12.6); a transient 

364 Discord error is surfaced but the token is still accepted so the rest of init 

365 can proceed (the bridge re-validates at boot anyway). 

366 """ 

367 existing = current.get("DISCORD_BOT_TOKEN", "") 

368 while True: 

369 pasted = prompter.secret( 

370 f"DISCORD_BOT_TOKEN {_set_label(existing)} — paste to set, enter to keep:" 

371 ) 

372 token = pasted or existing 

373 if not token: 

374 return "" 

375 try: 

376 identity = verify_identity_fn(token) 

377 except discord_discovery.DiscordAuthError: 

378 # The token Discord rejected is unusable; clear the kept value so an 

379 # empty answer can't "keep" the bad one, and ask for a fresh paste. 

380 print(" token rejected by Discord — paste a fresh bot token.") 

381 existing = "" 

382 continue 

383 except discord_discovery.DiscordDiscoveryError as e: 

384 # Couldn't reach Discord / rate-limited: don't block setup on a blip. 

385 print(f" could not verify token right now ({e}); continuing — the bridge will re-check.") 

386 if pasted: 386 ↛ 388line 386 didn't jump to line 388 because the condition on line 386 was always true

387 _envfile.upsert(env_path, {"DISCORD_BOT_TOKEN": pasted}) 

388 return token 

389 print(f" Connected as {identity.username} (id {identity.id}).") 

390 if pasted: 

391 _envfile.upsert(env_path, {"DISCORD_BOT_TOKEN": pasted}) 

392 return token

508 prompter: Prompter, 

509 *, 

510 name: str, 

511 home: Path | None, 

512 agents_dir: Path, 

513 env_path: Path, 

514 server_urls: str, 

515 start_fn: Callable[..., Awaitable[int]] | None, 

516 agent_start_fn: Callable[..., Awaitable[int]] | None, 

517 first_reply_fn: Callable[..., Awaitable[bool]] | None, 

518 pc_binary_fn: Callable[[], str] | None, 

519) -> int: 

520 """The ends-live finish (§4.6 / §12.6): start substrate → agent → watch reply. 

521 

522 Only possible on a **native install** (the supervisor is install-scoped — its 

523 lock, derived REST port, generated YAML, logs, and shim launcher all live 

524 under ``$CALFCORD_HOME``). On a dev run (``home is None``) or when the 

525 process-compose binary is missing, this DEGRADES to honest manual next-steps 

526 instead of orchestrating something it cannot (no green light that lies). 

527 

528 On the native happy path it composes :func:`lifecycle.start` → 

529 :func:`roster.agent_start` → :func:`wait_for_first_reply` (started FIRST so 

530 its ``latest``-offset group has joined) → an in-flow ``@<name> hello`` prompt 

531 once the watcher is listening, mapping each failure to its specific hint: 

532 

533 * substrate not ready → tear-down already happened in ``start``; map to the 

534 "privileged intents are probably off" hint and stop (don't clock the agent 

535 into a workspace that isn't up); 

536 * agent start failed → stop before the reply watch; 

537 * reply detected → 🎉; reply timed out → the bounded "org is live — try it 

538 yourself / run ``calfcord doctor``" downgrade. 

539 """ 

540 pc_binary_fn = pc_binary_fn or _default_pc_binary 

541 

542 if home is None or not _supervisor_available(pc_binary_fn): 

543 _print_manual_finish(name) 

544 return 0 

545 

546 # Resolve the real orchestration coroutines lazily (import-light): the agent 

547 # deployment path must not pull supervisor/control-plane modules at import. 

548 if start_fn is None or agent_start_fn is None or first_reply_fn is None: 

549 from calfcord.control_plane.first_reply import wait_for_first_reply 

550 from calfcord.supervisor import lifecycle, roster 

551 

552 start_fn = start_fn or lifecycle.start 

553 agent_start_fn = agent_start_fn or roster.agent_start 

554 first_reply_fn = first_reply_fn or wait_for_first_reply 

555 

556 return asyncio.run( 

557 _finish_live( 

558 prompter, 

559 name=name, 

560 home=home, 

561 agents_dir=agents_dir, 

562 server_urls=server_urls, 

563 start_fn=start_fn, 

564 agent_start_fn=agent_start_fn, 

565 first_reply_fn=first_reply_fn, 

566 ) 

567 )

570async def _finish_live( 

571 prompter: Prompter, 

572 *, 

573 name: str, 

574 home: Path, 

575 agents_dir: Path, 

576 server_urls: str, 

577 start_fn: Callable[..., Awaitable[int]], 

578 agent_start_fn: Callable[..., Awaitable[int]], 

579 first_reply_fn: Callable[..., Awaitable[bool]], 

580) -> int: 

581 """Run the native live finish; returns 0 on a live org, non-zero on a failure 

582 *before* the org could be reached (substrate / agent start).""" 

583 from calfcord.cli._agents import detect_agents 

584 

585 print("Opening your workspace (broker + bridge)…") 

586 # Mirror main.py's _run_lifecycle wiring (DRY): the shim launcher every 

587 # supervised process execs under, the broker URL, the defined roster, and 

588 # the mcp.json servers. Unlike `calfcord start`, a broken mcp.json is a 

589 # WARNING here: onboarding's job is reaching a live org, and MCP slots are 

590 # optional — the strict readers surface the error for fixing afterwards. 

591 from calfcord.mcp.config import McpConfigError, list_server_names, resolve_config_path 

592 

593 try: 

594 mcp_servers = list_server_names(resolve_config_path()) 

595 except McpConfigError as exc: 

596 print(f" warning: skipping MCP servers ({exc})") 

597 mcp_servers = [] 

598 launcher = str(home / "shims" / "calfcord") 

599 rc = await start_fn( 

600 home, 

601 server_urls=server_urls, 

602 launcher=launcher, 

603 agent_ids=detect_agents(agents_dir), 

604 mcp_servers=mcp_servers, 

605 ) 

606 if rc != 0: 

607 # start() already tore the substrate down and printed the specific cause; 

608 # map a readiness failure to the most-missed onboarding cause (§12.6). 

609 print( 

610 " the workspace didn't come up. If the broker is running, the bridge's " 

611 "privileged intents are probably off — enable Message Content and Server " 

612 "Members on the Bot tab, then re-run `calfcord init`." 

613 ) 

614 return rc 

615 

616 print(f"Bringing {name} online…") 

617 rc = await agent_start_fn(home, name=name, server_urls=server_urls) 

618 if rc != 0: 

619 return rc 

620 

621 # In-flow first-reply prompt (§12.6: prompt the @mention INSIDE init, fixing 

622 # the old step3/step4 contradiction). The wizard sends nothing — the human 

623 # posts it; we only consume the outbox reply. 

624 # 

625 # Start the watcher FIRST and wait until its ``latest``-offset consumer group 

626 # has JOINED before prompting: otherwise a fast human posts ``@<agent> hello`` 

627 # before the group joins, the reply lands before we are subscribed, and we 

628 # wrongly downgrade on a live org. The watcher runs as a task so its 

629 # ``worker.start()`` (the group join) makes progress while we await its 

630 # readiness; ``_FIRST_REPLY_TIMEOUT_S`` still bounds the whole wait. 

631 watcher_ready = asyncio.Event() 

632 watch_task = asyncio.ensure_future( 

633 first_reply_fn( 

634 server_urls, agent_id=name, timeout_s=_FIRST_REPLY_TIMEOUT_S, ready=watcher_ready 

635 ) 

636 ) 

637 # Proceed to the prompt the instant EITHER the watcher reports joined OR the 

638 # task finishes first. Racing against task completion means a watcher that 

639 # returns/raises before ever signalling ready (broker unreachable, an early 

640 # error) does not hang the wizard waiting for a readiness that will never 

641 # come — we fall through to the prompt and let ``await watch_task`` surface 

642 # the (already-resolved) result or exception. 

643 ready_wait = asyncio.ensure_future(watcher_ready.wait()) 

644 try: 

645 await asyncio.wait({ready_wait, watch_task}, return_when=asyncio.FIRST_COMPLETED) 

646 finally: 

647 # Cancel and drain the readiness waiter (it may still be pending if the 

648 # task completed first) so it is never left as an un-retrieved pending 

649 # task — its CancelledError is expected and swallowed. 

650 ready_wait.cancel() 

651 with contextlib.suppress(asyncio.CancelledError): 

652 await ready_wait 

653 prompter.confirm(f"In Discord, say: @{name} hello — press enter once you've sent it.", default=True) 

654 print("Watching for the first reply…") 

655 # The org is ALREADY live (substrate + agent both started). First-reply 

656 # detection is advisory, and the watcher opens its OWN Client.connect / 

657 # worker.start — a broker drop or transient connect blip mid-watch raises out 

658 # of wait_for_first_reply (which only swallows TimeoutError). Don't let that 

659 # crash the wizard after the org came up: degrade any non-timeout failure into 

660 # the same bounded "org is live — try it yourself / run doctor" fallback below. 

661 # ``except Exception`` (not bare) deliberately lets asyncio.CancelledError 

662 # propagate — swallowing the rest is correct here: the failure is in detection, 

663 # not in the (already-live) org. 

664 try: 

665 detected = await watch_task 

666 except Exception: 

667 detected = False 

668 if detected: 

669 print(f"🎉 {name} replied — your organization is live!") 

670 else: 

671 # Bounded fallback (§12.6): never promise more than we detected. 

672 print( 

673 f" your organization is live — try `@{name} hello` in Discord. " 

674 "If nothing replies, run `calfcord doctor`." 

675 ) 

676 print() 

677 print(f"({_REBOOT_NOTE} `calfcord start` reopens it; `calfcord status` shows who's online.)") 

678 return 0
