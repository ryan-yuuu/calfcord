1"""``calfcord router show|set|edit`` + ``router start|stop`` — the router's 

2first-class, editable config surface and its lifecycle (design §12.0). 

3 

4The router holds an LLM connection like an agent, so — unlike a one-shot wizard — 

5it gets an *editable* config surface that mirrors the agent config ergonomics: 

6 

7* ``show`` renders the current provider/model (never any secret), like 

8 ``agent show`` renders a definition's fields; 

9* ``set`` writes provider/model non-interactively, validated the same way an 

10 agent's provider is (an unknown provider is rejected, nothing is persisted); 

11* ``edit`` runs the interactive provider sub-flow shared by every wizard 

12 (:func:`calfcord.cli._providers.configure_provider`). 

13 

14This module **reconciles the old one-shot ``router setup`` into ONE editable 

15path**: ``edit`` is the wizard ``router setup`` used to be (same explanation, same 

16``configure_provider`` sub-flow, same ``cheap=True`` bias for a per-message 

17classifier), and ``show`` / ``set`` are the non-interactive surfaces it lacked. 

18 

19Config persists to the two ``CALFKIT_ROUTER_*`` env vars the router runner already 

20reads (:func:`calfcord.router.definition.build_router_definition`), so a running 

21router picks up a change on its next (re)start with no code change. The var names 

22are imported from :mod:`calfcord.router.definition` so producer and consumer 

23cannot drift. 

24 

25Lifecycle (``router start|stop``) is built on the generic 

26:func:`calfcord.supervisor.component.component_start` / 

27:func:`~calfcord.supervisor.component.component_stop` (DRY with tools), with 

28one router-specific rule: **``router start`` FAILS FAST when unconfigured** — it 

29refuses to launch a router with no provider/model *before* any supervisor call, so 

30the operator gets an actionable "configure it first" message instead of a process 

31that boots and immediately dies on a missing LLM target. 

32 

33This is **not** SDK-light: ``from calfcord.agents.definition import Provider`` triggers 

34the ``calfcord.agents`` package ``__init__``, which eagerly imports 

35``agents.factory`` and through it the anthropic/openai provider SDKs. We accept 

36that transitive cost — the entry point already pays it for the agent verbs — and 

37note it here so nobody mistakes this module for a lazy-import seam it is not. 

38"""

71def _router_provider_model(env: dict[str, str]) -> tuple[str | None, str | None]: 

72 """Read the configured ``(provider, model)`` from a parsed ``.env``. 

73 

74 Empty strings are normalized to ``None`` so this agrees with the runner's 

75 ``or`` chain (:func:`build_router_definition`), which treats an empty env var 

76 as "unset" and falls through to the bundled ``router.md`` / in-code default. 

77 """ 

78 provider = env.get(_PROVIDER_VAR) or None 

79 model = env.get(_MODEL_VAR) or None 

80 return provider, model

83def show(*, env_path: Path) -> int: 

84 """``calfcord router show``: print the router's current provider/model. 

85 

86 Reads only the two ``CALFKIT_ROUTER_*`` vars and renders them — never any API 

87 key that shares the file, so this is safe to run anywhere. An unconfigured 

88 router prints an explicit "(not configured)" line (and how to fix it) rather 

89 than two blanks. Always returns ``0`` — ``show`` is read-only and 

90 "not configured yet" is a valid state, not an error. 

91 """ 

92 provider, model = _router_provider_model(read_env(env_path)) 

93 

94 if provider is None and model is None: 

95 print("router: (not configured)") 

96 print("Configure it with: calfcord router set --provider <p> --model <m>") 

97 print(" or: calfcord router edit (interactive)") 

98 return 0 

99 

100 print(f"router provider: {provider or '(not set)'}") 

101 print(f"router model: {model or '(not set)'}") 

102 return 0

105def set_config(*, env_path: Path, provider: str | None, model: str | None) -> int: 

106 """``calfcord router set [--provider P] [--model M]``: write the router LLM config. 

107 

108 A partial set is honoured (write only the flag(s) given; leave the other var 

109 untouched), so an operator can retarget the model without restating the 

110 provider. The provider is validated against the same 

111 :data:`~calfcord.agents.definition.Provider` tags the runner accepts — an 

112 unknown provider is rejected and **nothing** is persisted (no half-write); a 

113 blank model is likewise rejected. ``set`` with neither flag is a usage error, 

114 not a silent no-op. 

115 

116 Returns ``0`` on a successful write, ``1`` on a validation/usage error (with an 

117 actionable message; the file is untouched on any rejection). 

118 """ 

119 if provider is None and model is None: 

120 print("error: nothing to set; pass --provider and/or --model.") 

121 return 1 

122 

123 updates: dict[str, str] = {} 

124 

125 if provider is not None: 

126 if provider not in _VALID_PROVIDERS: 

127 print( 

128 f"error: unknown provider {provider!r}; " 

129 f"choose one of {', '.join(_VALID_PROVIDERS)}." 

130 ) 

131 return 1 

132 updates[_PROVIDER_VAR] = provider 

133 

134 if model is not None: 

135 # A model is a free-form slug the provider accepts, but an *empty* one 

136 # would persist a meaningless value the runner reads as "unset" — reject 

137 # it loudly rather than write a blank. 

138 if not model.strip(): 

139 print("error: --model must be a non-empty model id.") 

140 return 1 

141 updates[_MODEL_VAR] = model 

142 

143 upsert(env_path, updates) 

144 print("router config updated:") 

145 if _PROVIDER_VAR in updates: 

146 print(f" provider -> {updates[_PROVIDER_VAR]}") 

147 if _MODEL_VAR in updates: 

148 print(f" model -> {updates[_MODEL_VAR]}") 

149 # The terse next-step block (behavior #3): a sentence ending in a colon, a 

150 # blank line, the two-space-indented command. The router bakes its config at 

151 # construction, so a config change takes effect via the roster `restart` verb. 

152 print("\nRestart the router to apply:\n\n calfcord router restart") 

153 return 0

156def edit(prompter: Prompter, *, env_path: Path) -> int: 

157 """``calfcord router edit``: the interactive router-config flow. 

158 

159 This is the path the old one-shot ``calfcord router setup`` wizard became — it 

160 explains the (optional, ambient) router, runs the shared provider sub-flow 

161 (:func:`_providers.configure_provider` with ``cheap=True`` biasing the model 

162 default toward a fast tier — the right call for a per-message classifier), and 

163 persists the choice as the two ``CALFKIT_ROUTER_*`` vars. Reads the current 

164 ``.env`` once so a re-run pre-selects the operator's existing provider/model 

165 and the provider sub-flow can show/keep already-set credentials. Returns ``0``. 

166 

167 The provider sub-flow owns all credential prompting and live model fetching; we 

168 persist only the resulting provider/model (never a secret) via the 

169 position-preserving :func:`upsert`. 

170 """ 

171 current = read_env(env_path) 

172 

173 print("calfcord router edit — configuring the OPTIONAL ambient router in", env_path) 

174 print() 

175 print("The router decides which agent answers a message that has NO @mention.") 

176 print("It runs an LLM call once per ambient message, so a fast/cheap model is") 

177 print("recommended. It is optional: without it, @mentions still route to agents") 

178 print("and un-@mentioned messages are simply left unanswered.") 

179 print() 

180 

181 # Default the provider to the router's own prior choice, then the agents' 

182 # default provider, then anthropic — so a re-run keeps the operator's pick and 

183 # a first run inherits the provider they already set up in ``init``. 

184 default_provider = ( 

185 current.get(_PROVIDER_VAR) or current.get(_AGENT_DEFAULT_PROVIDER_VAR) or "anthropic" 

186 ) 

187 provider, model = _providers.configure_provider( 

188 prompter, 

189 env_path=env_path, 

190 current=current, 

191 default_provider=default_provider, 

192 cheap=True, 

193 current_model=current.get(_MODEL_VAR), 

194 ) 

195 upsert(env_path, {_PROVIDER_VAR: provider, _MODEL_VAR: model}) 

196 

197 print() 

198 print(f"Router will use {provider}/{model}.") 

199 print("It is optional — skip it and @mentions still work.") 

200 # The terse next-step block (behavior #3): a sentence ending in a colon, a 

201 # blank line, the two-space-indented command. The router bakes its config at 

202 # construction, so a config change takes effect via the roster `restart` verb 

203 # (which also brings a stopped router up — so it is the right steer whether or 

204 # not the router was already running). 

205 print("\nRestart the router to apply:\n\n calfcord router restart") 

206 return 0

209def _is_configured(env_path: Path) -> bool: 

210 """Whether BOTH router vars are set (non-empty) in ``env_path``. 

211 

212 The fail-fast precondition for ``router start``: the runner needs both a 

213 provider and a model (a half-configured router boots and dies), and an empty 

214 string counts as unset to match the runner's ``or`` chain. 

215 """ 

216 provider, model = _router_provider_model(read_env(env_path)) 

217 return provider is not None and model is not None

220async def router_start( 

221 home: str | os.PathLike[str], 

222 *, 

223 env_path: Path, 

224 client=None, 

225) -> int: 

226 """``calfcord router start``: bring the router online — but only if configured. 

227 

228 **Fail-fast precondition (design §12.0):** if the router has no provider/model 

229 the runner would boot and immediately die, so refuse here — *before any 

230 supervisor call* — and point the operator at ``router set`` / ``router edit``. 

231 Only once configured do we delegate to the generic 

232 :func:`calfcord.supervisor.component.component_start` (the same DRY base 

233 tools uses), which does the workspace check and starts the ``router`` slot. 

234 

235 ``client`` is injected for testing; production defaults it (in 

236 ``component_start``) to a per-home REST client. Returns ``0`` on a successful 

237 start, ``1`` when unconfigured or the workspace is down. 

238 """ 

239 if not _is_configured(env_path): 

240 print( 

241 "error: the router is not configured (needs a provider and model). " 

242 "Set it first: calfcord router set --provider <p> --model <m> " 

243 "(or run: calfcord router edit)." 

244 ) 

245 return 1 

246 

247 return await component_start(home, name=_ROUTER_PROCESS_NAME, client=client)

265async def router_restart( 

266 home: str | os.PathLike[str], 

267 *, 

268 client=None, 

269) -> int: 

270 """``calfcord router restart``: reload the running router after a config change. 

271 

272 The apply mechanism behind ``router set`` / ``router edit``'s next-step hint: 

273 the runner bakes its provider/model at construction, so a restart is how a 

274 config edit takes effect on a live router. Like ``stop`` it runs NO config 

275 precheck — a running router already had valid config, and ``component_restart`` 

276 issues the REST restart unconditionally — so it delegates straight to 

277 :func:`calfcord.supervisor.component.component_restart` for the ``router`` slot. 

278 ``client`` is injected for testing. Returns ``0`` on success, ``1`` when the 

279 workspace is down. 

280 """ 

281 return await component_restart(home, name=_ROUTER_PROCESS_NAME, client=client)
