1"""Web tools: ``web_fetch`` and ``web_search``. 

2 

3Thin wrappers around ``smolagents``' :class:`VisitWebpageTool` and 

4:class:`DuckDuckGoSearchTool`. Each tool is constructed lazily as a 

5module-global singleton so import-time cost stays low — the upstream 

6classes pull network adapters on first call. 

7 

8* ``web_fetch`` — fetches a URL, converts HTML to markdown, returns the 

9 text. Smolagents handles encoding, timeout, and markdown conversion. 

10 No API key needed. 

11* ``web_search`` — DuckDuckGo via the ``ddgs`` library. No API key. 

12 If you want richer results (Brave, Tavily, SerpAPI), see 

13 ``docs/authoring-tools.md`` for the swap path. 

14 

15Both ``VisitWebpageTool`` (``markdownify``) and ``DuckDuckGoSearchTool`` 

16(``ddgs``) need third-party packages that smolagents ships ONLY behind 

17its optional ``toolkit`` extra — bare ``smolagents`` does not pull them 

18in. This project therefore depends on ``smolagents[toolkit]``; without 

19the extra, the first ``web_search`` call raises ``ImportError`` ("You 

20must install package ``ddgs`` ...") from the tool constructor. 

21 

22Both wrappers convert smolagents exceptions to ``"error: ..."`` strings 

23rather than raising, so the calling LLM can adapt (retry, rephrase 

24query, give up) instead of triggering the tool retry-with-feedback path 

25on a transient network error. 

26"""

43_MAX_FETCH_CHARS = 50_000 

44"""Defense-in-depth cap on what ``web_fetch`` returns. Smolagents already 

45truncates internally; this guard catches a misconfigured local mirror or 

46an article whose markdown body slips past the upstream limit. The unit 

47is *characters* (not bytes — non-Latin scripts use multi-byte UTF-8 but 

48single chars). 50K is roughly a long article's worth of English 

49markdown; for non-Latin or dense code dumps the same char budget yields 

50fewer tokens, which keeps the LLM input safe across content types.""" 

51

73async def web_fetch(ctx: ToolContext, url: str) -> str: 

74 """Fetch a web page and return its content as markdown. 

75 

76 Use this when the user mentions a URL, when you need to read API 

77 docs / a blog post / a README, or to follow up on a ``web_search`` 

78 result. HTML is converted to markdown so headings, links, and lists 

79 survive the round-trip; images and styling do not. 

80 

81 Args: 

82 url: A fully-qualified ``http://`` or ``https://`` URL. 

83 

84 Returns: 

85 The page content as markdown, capped at 50KB. On network errors 

86 (DNS failure, 4xx/5xx, timeout) returns an ``"error: ..."`` 

87 message so the caller can adapt. 

88 """ 

89 _ = ctx 

90 try: 

91 result = _get_visit_tool().forward(url=url) # type: ignore[attr-defined] 

92 except Exception as e: 

93 # Catch broad: smolagents wraps a wide variety of network/parse 

94 # errors and we want all of them to flow back to the LLM as a 

95 # recoverable string rather than triggering tool-error retry. 

96 logger.warning("web_fetch failed url=%s: %s", url, e) 

97 return f"error: web_fetch failed for {url!r}: {e}" 

98 if not isinstance(result, str): 98 ↛ 99line 98 didn't jump to line 99 because the condition on line 98 was never true

99 result = str(result) 

100 if len(result) > _MAX_FETCH_CHARS: 

101 return result[:_MAX_FETCH_CHARS] + f"\n\n(truncated at {_MAX_FETCH_CHARS} chars)" 

102 return result
