1"""``calfcord deploy <systemd|k8s|docker>`` — Altitude-3 graduation manifests. 

2 

3calfcord's day-one supervisor is Process Compose (single host, generated YAML the 

4operator never edits). ``deploy`` is the *graduation* surface (design §2 / §6 / 

5§11.5): it emits the heavier-tier artifacts an operator hand-validates before 

6running across boxes. The portable invariant is the **Kafka wire + ``.env`` + 

7``agents/*.md``**, NOT the Process Compose project — so these manifests re-derive 

8the same roster from the same seam (:func:`detect_agents`) and dial a shared 

9broker, exactly like the distributed primitive (§12.5). 

10 

11Three targets, each with a different honesty posture (§11.6): 

12 

13* **systemd** — a *real, correct-by-construction* unit for the single-host 

14 substrate. ``calfcord start`` forks a detached supervisor and returns 0 once the 

15 bridge is healthy (findings.start_model), which is precisely ``Type=forking``; 

16 the unit runs the install **shim** (``<shim> start`` / ``<shim> stop``) — the 

17 single seam that owns the ``up`` flags, the derived REST port, the priming 

18 reconcile and the readiness gate — never a reconstructed ``up`` argv (§12.3 / 

19 §13.2). It is the one artifact whose correctness we can vouch for, yet it is 

20 still headed as a reference because per-host paths/users vary. 

21* **k8s** — *reference* manifests (clearly annotated): a broker workload, a 

22 ConfigMap with the shared ``CALF_HOST_URL``, and one Deployment per process type 

23 (bridge / router / tools) plus one per *defined* agent, all on the shipped 

24 calfcord image running the ``calfkit-*`` console scripts the compose uses. NOT 

25 ``calfcord start`` (there is no in-pod supervisor); each process type is its own 

26 workload dialing the shared broker — the Altitude-3 distributed shape. Secrets 

27 arrive via a ``Secret`` reference, never inlined (§12.3). 

28* **docker** — the shipped ``docker-compose.yml`` is hand-tuned (Codex auth 

29 mounts, the A2A channel override, ``depends_on`` healthchecks); regenerating it 

30 would lose that nuance, so ``deploy docker`` *points at the real file* and emits 

31 an optional per-agent ``compose.override.yml`` snippet (``calfkit-agent 

32 <name>``) for crash isolation, derived from the roster. 

33 

34Decoupling invariant (§12.3): this module renders text from a roster + paths and 

35inlines **no secret literal**; it imports only :func:`detect_agents` / 

36:func:`read_env` (schema/path-only seams). ``deploy`` 

37reads the install's agents + ``.env`` off disk and never talks to the running 

38supervisor — so it needs no REST probe and no running supervisor to render. It 

39does, however, require a native install (``CALFCORD_HOME``): the emitted manifests 

40reference the install's shim launcher and home paths, so the :mod:`main` veneer 

41(:func:`_run_deploy`) refuses on a dev tree rather than rendering a manifest that 

42points at nothing. 

43"""

82def _resolve_config_host_url(server_urls: str) -> str: 

83 """Resolve the ConfigMap ``CALF_HOST_URL`` for the bundled-broker manifest. 

84 

85 The default deploy path passes ``CALF_HOST_URL or "localhost"`` (single host), 

86 but the k8s reference ships a Service named ``broker`` and one pod per process 

87 type: a verbatim ``localhost`` would make every pod dial its own loopback and 

88 the manifest fails on apply. So a loopback host (``localhost`` / ``127.0.0.1`` / 

89 ``::1`` / empty, with or without ``:PORT``) resolves to the in-cluster Service 

90 name, preserving an explicit port (a custom loopback port is the broker's 

91 listener port). A real external/non-loopback broker passes through verbatim. 

92 """ 

93 candidate = server_urls.strip() 

94 # A multi-broker bootstrap list (e.g. "h1:9092,h2:9092") is the operator's 

95 # explicit cross-host target — not a single loopback to rewrite — and is not a 

96 # single URL ``urlsplit`` can parse, so pass it through verbatim. 

97 if "," in candidate: 

98 return server_urls 

99 # The bare (unbracketed) IPv6 loopback does not survive ``urlsplit`` as a 

100 # ``.hostname`` (only the bracketed "[::1]" form does), so normalise it here; 

101 # a bare "::1" carries no port, so it resolves to the broker's default. 

102 if candidate.lower() == "::1": 

103 return f"{_K8S_BROKER_SERVICE}:{_K8S_BROKER_PORT}" 

104 # urlsplit needs a scheme to populate .hostname/.port; the wire form is bare 

105 # host:port (e.g. "localhost:9092"), so prepend a dummy scheme to parse it. 

106 # ``.port`` raises ValueError on a non-integer/out-of-range port; an 

107 # unparseable value is not a loopback we can confidently rewrite, so fall back 

108 # to the verbatim passthrough rather than crash ``calfcord deploy``. 

109 try: 

110 parsed = urlsplit(f"//{candidate}") 

111 host = (parsed.hostname or "").lower() 

112 port = parsed.port or _K8S_BROKER_PORT 

113 except ValueError: 

114 return server_urls 

115 if host not in _LOOPBACK_HOSTS: 

116 return server_urls 

117 return f"{_K8S_BROKER_SERVICE}:{port}"

135def render_systemd(*, home: str, launcher: str) -> str: 

136 """Render a systemd unit for the single-host substrate. 

137 

138 Models how ``calfcord start`` actually launches the supervisor: it forks a 

139 detached process-compose that outlives the CLI and returns 0 once the bridge 

140 is healthy — exactly the ``Type=forking`` contract (findings.start_model). The 

141 unit runs the install **shim** (``<launcher> start`` / ``<launcher> stop``), 

142 the single seam that knows the ``up`` flags, the home-derived REST port, the 

143 priming reconcile and the readiness gate; reconstructing the ``up`` argv here 

144 would duplicate the §13.2 contract and drift. ``CALFCORD_HOME`` is exported so 

145 the shim and ``pc_port_for`` resolve the same home. ``Restart=on-failure`` only 

146 reacts to the *fork* (the ``calfcord start`` ExecStart) exiting non-zero — with 

147 ``Type=forking`` and no ``PIDFile=`` systemd cannot track the detached 

148 process-compose, so a crash *inside* the supervised tree is not auto-recovered 

149 here (process-compose's own per-process restarts cover that); an 

150 operator-commanded ``stop`` is left alone. 

151 

152 Emitted as a *user* unit (``WantedBy=default.target``, ``systemctl --user``): 

153 the supervisor runs under the install owner's session, so the unit carries no 

154 system-only ``User=`` / ``Group=`` directives (a ``--user`` unit rejects them). 

155 Headed as a *reference* unit (§11.6): the shape is correct, but per-host paths 

156 vary, so the operator validates before enabling. 

157 """ 

158 # Plain text (not configparser.write, which lowercases nothing but emits its 

159 # own quirks): a systemd unit is hand-readable and the operator edits it. 

160 return ( 

161 "# calfcord substrate — systemd USER unit (REFERENCE: validate paths for your host).\n" 

162 "#\n" 

163 "# Models `calfcord start`: it forks a detached process-compose supervisor and\n" 

164 "# returns 0 once the bridge is healthy, so Type=forking is the faithful type.\n" 

165 "# ExecStart/ExecStop run the install shim — the single seam that owns the\n" 

166 "# process-compose `up` flags, the home-derived REST port, and the readiness gate.\n" 

167 "# Install for the current login session: systemctl --user enable --now calfcord\n" 

168 "# (a --user unit already runs as that login user — system-only owner directives\n" 

169 "# do not apply and are intentionally omitted).\n" 

170 "# Restart=on-failure only catches the `start` fork exiting non-zero; with\n" 

171 "# Type=forking and no PIDFile= systemd can't track the detached supervisor, so a\n" 

172 "# crash inside the tree is handled by process-compose's per-process restarts.\n" 

173 "[Unit]\n" 

174 "Description=calfcord substrate (broker + bridge)\n" 

175 "After=network-online.target\n" 

176 "Wants=network-online.target\n" 

177 "\n" 

178 "[Service]\n" 

179 "Type=forking\n" 

180 f"Environment=CALFCORD_HOME={home}\n" 

181 f"WorkingDirectory={home}\n" 

182 f"ExecStart={launcher} start\n" 

183 f"ExecStop={launcher} stop\n" 

184 "Restart=on-failure\n" 

185 "RestartSec=5\n" 

186 "\n" 

187 "[Install]\n" 

188 "WantedBy=default.target\n" 

189 )

233def render_k8s( 

234 *, 

235 agent_ids: list[str], 

236 server_urls: str, 

237 image: str = _DEFAULT_IMAGE, 

238 mcp_servers: list[str] | None = None, 

239) -> str: 

240 """Render *reference* Kubernetes manifests for a distributed calfcord. 

241 

242 A multi-document YAML stream: a comment header, a ConfigMap holding the shared 

243 ``CALF_HOST_URL`` (an external ``server_urls`` flows through verbatim; a 

244 localhost-ish one is rewritten to the bundled broker Service so pods don't dial 

245 their own loopback), a broker Service + Deployment, and one Deployment per 

246 process type (bridge / router / tools) plus one per *defined* agent — 

247 each running a ``calfkit-*`` console script on the shipped image, dialing the 

248 shared broker. This is the Altitude-3 distributed shape, NOT ``calfcord start`` 

249 (no in-pod supervisor). The roster is sorted so the document order never 

250 depends on ``.md`` glob order. 

251 

252 Annotated as a *reference* starting point (§11.6): cluster specifics 

253 (storage class for a durable broker, resource limits, ingress, the Secret's 

254 creation) are left to the operator. Secrets arrive via a ``Secret`` reference, 

255 never inlined (§12.3). 

256 """ 

257 roster = sorted(agent_ids) 

258 

259 header = ( 

260 "# calfcord — REFERENCE Kubernetes manifests (Altitude-3 distributed deploy).\n" 

261 "# A starting point to validate per cluster — NOT a turnkey production install.\n" 

262 "# Each process type is its own Deployment dialing the shared broker (no in-pod\n" 

263 "# supervisor; this is the distributed primitive, not `calfcord start`).\n" 

264 "#\n" 

265 "# Before applying, create the secrets the bridge/agents need (out of band):\n" 

266 f"# kubectl create secret generic {_K8S_SECRET_NAME} --from-env-file=.env\n" 

267 "# Tune replicas/resources/storage and swap the broker for your managed Kafka\n" 

268 "# (then drop the bundled broker workload and point CALF_HOST_URL at it).\n" 

269 "# Note: agents/*.md frontmatter the agent rewrites at runtime (e.g. thinking-\n" 

270 "# effort) is NON-PERSISTENT here — these pods get envFrom only, no writable\n" 

271 "# agents/ volume; mount a writable agents/ volume (PVC) if those edits must\n" 

272 "# survive a pod restart.\n" 

273 ) 

274 

275 # Resolve the shared CALF_HOST_URL once: an external/managed broker flows 

276 # through verbatim; a localhost-ish server_urls (the default _run_deploy hands 

277 # down) is rewritten to the bundled broker Service below — otherwise every pod 

278 # would dial its own loopback and the manifest would fail on apply. 

279 config_host_url = _resolve_config_host_url(server_urls) 

280 # The port the in-cluster roster dials for the bundled broker. When the resolved 

281 # URL targets our Service (localhost-ish input), the bundled broker Service + 

282 # advertised listener follow that port so the reference manifest is internally 

283 # consistent (a custom loopback port survives end-to-end). For an external 

284 # broker the bundled workload is moot (the header tells operators to drop it). 

285 cluster_broker_port = _K8S_BROKER_PORT 

286 if config_host_url.startswith(f"{_K8S_BROKER_SERVICE}:"): 

287 cluster_broker_port = int(config_host_url.rsplit(":", 1)[1]) 

288 

289 docs: list[dict] = [] 

290 

291 # Shared, non-secret config. 

292 docs.append( 

293 { 

294 "apiVersion": "v1", 

295 "kind": "ConfigMap", 

296 "metadata": {"name": "calfcord-config", "labels": {"app": "calfcord"}}, 

297 "data": {"CALF_HOST_URL": config_host_url}, 

298 } 

299 ) 

300 

301 # Broker: a single-instance workload + a ClusterIP Service the roster dials. 

302 # Memory storage (matching the shipped compose) — annotate durable storage as 

303 # an operator choice in the header. The container always listens on the broker's 

304 # native 9092; the Service maps the dialed port to it and the advertised listener 

305 # matches what clients dial (the resolved CALF_HOST_URL above). 

306 docs.append( 

307 { 

308 "apiVersion": "apps/v1", 

309 "kind": "Deployment", 

310 "metadata": { 

311 "name": _K8S_BROKER_SERVICE, 

312 "labels": {"app": "calfcord", "component": "broker"}, 

313 }, 

314 "spec": { 

315 "replicas": 1, 

316 "selector": {"matchLabels": {"app": "calfcord", "component": "broker"}}, 

317 "template": { 

318 "metadata": {"labels": {"app": "calfcord", "component": "broker"}}, 

319 "spec": { 

320 "containers": [ 

321 { 

322 "name": "broker", 

323 "image": "ghcr.io/tansu-io/tansu:latest", 

324 "args": [ 

325 "broker", 

326 "--storage-engine=memory://tansu/", 

327 f"--listener-url=tcp://0.0.0.0:{_K8S_BROKER_PORT}", 

328 "--advertised-listener-url=" 

329 f"tcp://{_K8S_BROKER_SERVICE}:{cluster_broker_port}", 

330 ], 

331 "ports": [{"containerPort": _K8S_BROKER_PORT}], 

332 } 

333 ] 

334 }, 

335 }, 

336 }, 

337 } 

338 ) 

339 docs.append( 

340 { 

341 "apiVersion": "v1", 

342 "kind": "Service", 

343 "metadata": { 

344 "name": _K8S_BROKER_SERVICE, 

345 "labels": {"app": "calfcord", "component": "broker"}, 

346 }, 

347 "spec": { 

348 "selector": {"app": "calfcord", "component": "broker"}, 

349 "ports": [{"port": cluster_broker_port, "targetPort": _K8S_BROKER_PORT}], 

350 }, 

351 } 

352 ) 

353 

354 # Fixed process-type Deployments (singletons), then one per defined agent, 

355 # then one per mcp.json server (per-server isolation, like the local slots). 

356 for component, script in _PROCESS_COMMANDS.items(): 

357 docs.append(_k8s_deployment(name=component, image=image, command=[script])) 

358 for server in mcp_servers or []: 

359 docs.append( 

360 _k8s_deployment( 

361 name=mcp_slot_name(server), 

362 image=image, 

363 command=["calfkit-mcp", server], 

364 ) 

365 ) 

366 for agent_id in roster: 

367 docs.append( 

368 _k8s_deployment( 

369 name=f"agent-{agent_id}", 

370 image=image, 

371 command=["calfkit-agent", agent_id], 

372 ) 

373 ) 

374 

375 body = yaml.safe_dump_all(docs, sort_keys=False, explicit_start=True) 

376 return f"{header}{body}"

382def render_docker(*, repo_compose_path: str, agent_ids: list[str]) -> str: 

383 """Point the operator at the shipped compose + an optional per-agent override. 

384 

385 The shipped ``docker-compose.yml`` is hand-tuned — Codex auth mounts, the A2A 

386 channel override, ``depends_on`` healthchecks — so regenerating it would lose 

387 that nuance. ``deploy docker`` therefore names the real file and, when the 

388 roster has agents, emits a ``compose.override.yml`` snippet that *replaces* the 

389 single all-in-one ``agent`` service with one ``calfkit-agent <name>`` service 

390 per defined agent (crash isolation: a crash in one agent's container no longer 

391 takes the others down). The roster is sorted so the snippet is deterministic. 

392 

393 The replacement is the load-bearing detail: an override that only *added* 

394 ``agent-<id>`` services would leave the base ``agent`` running too (it boots 

395 *all* agents in one process), so ``docker compose up`` would double-run every 

396 agent. Compose merges an override onto the base service-by-service, so the 

397 snippet pins ``agent: {scale: 0}`` — which ``docker compose up`` honours by 

398 starting 0 containers of the base — turning the additive merge into a genuine 

399 split. 

400 """ 

401 roster = sorted(agent_ids) 

402 

403 lines = [ 

404 "# calfcord — Docker deployment.", 

405 "#", 

406 "# The shipped docker-compose.yml is hand-tuned (Codex auth mounts, the A2A", 

407 "# channel override, depends_on healthchecks) — do NOT regenerate it. Use it", 

408 "# directly:", 

409 f"# docker compose -f {repo_compose_path} up --build", 

410 "#", 

411 ] 

412 

413 if not roster: 

414 lines.append("# No agents are defined yet — `calfcord agent create` first, then re-run.") 

415 return "\n".join(lines) + "\n" 

416 

417 lines += [ 

418 "# For per-agent crash isolation, drop this compose.override.yml next to the", 

419 "# shipped file: it REPLACES the single all-in-one `agent` service with one", 

420 "# service per defined agent (each runs `calfkit-agent <name>`), so one agent", 

421 "# crashing no longer restarts the others. The base `agent` is disabled with", 

422 "# `scale: 0` (compose starts 0 of it) so it does not double-run every agent", 

423 "# alongside the per-agent services. Each per-agent service inherits the base", 

424 "# build/image/env via `extends` — keep the Codex mounts/CALF_HOST_URL there.", 

425 "#", 

426 "# --- compose.override.yml ---", 

427 "services:", 

428 # Disable the all-in-one base service: compose merges this override onto the 

429 # base `agent` (it does not remove it), so without scale: 0 `docker compose 

430 # up` would run the base (all agents in one) PLUS every agent-<id> below. 

431 " agent:", 

432 " scale: 0", 

433 ] 

434 for agent_id in roster: 

435 # Each per-agent service only sets `command:`; everything else (build, 

436 # image, env_file, CALF_HOST_URL, Codex mounts) is inherited from the base 

437 # `agent` service in the shipped file via `extends`, so secrets/mounts are 

438 # never duplicated here (and no secret is inlined, §12.3). 

439 lines += [ 

440 f" agent-{agent_id}:", 

441 " extends:", 

442 " file: docker-compose.yml", 

443 " service: agent", 

444 f" command: calfkit-agent {agent_id}", 

445 ] 

446 

447 return "\n".join(lines) + "\n"

453def run( 

454 target: str, 

455 *, 

456 home: Path, 

457 env_path: Path, 

458 agents_dir: Path, 

459 server_urls: str, 

460 out_path: Path | None = None, 

461) -> int: 

462 """Render ``target``'s manifest to stdout (or ``out_path``) and return a code. 

463 

464 The thin veneer over the pure render functions: it resolves the launcher 

465 (``<home>/shims/calfcord`` — the same shim every supervised ``command`` is 

466 built on), enumerates the roster via :func:`detect_agents` (the exact seam 

467 ``calfcord start`` uses, so the manifest's roster equals what ``start`` 

468 declares), and reads ``.env`` only to *preflight* (never to inline secret 

469 values). Output goes to stdout by default; ``out_path`` writes the manifest to 

470 a file instead. An unknown ``target`` returns non-zero with an error rather 

471 than rendering garbage (argparse ``choices=`` guards the CLI, but a direct 

472 caller must fail cleanly too). 

473 """ 

474 if target not in _VALID_TARGETS: 

475 print(f"error: unknown deploy target {target!r} (choose one of {', '.join(_VALID_TARGETS)})") 

476 return 1 

477 

478 roster = detect_agents(agents_dir) 

479 launcher = str(home / "shims" / "calfcord") 

480 

481 if target == "systemd": 

482 # Preflight: a substrate unit only makes sense once a broker is configured. 

483 # Read .env for the surfaced hint — never inline its values. 

484 if not read_env(env_path).get("CALF_HOST_URL"): 484 ↛ 489line 484 didn't jump to line 489 because the condition on line 484 was always true

485 print( 

486 "# note: CALF_HOST_URL is not set in this install's .env yet — the substrate " 

487 "won't start until it is (run `calfcord init` or `calfcord self set-broker`)." 

488 ) 

489 manifest = render_systemd(home=str(home), launcher=launcher) 

490 elif target == "k8s": 

491 mcp_servers = configured_mcp_servers_or_none() 

492 if mcp_servers is None: 492 ↛ 493line 492 didn't jump to line 493 because the condition on line 492 was never true

493 return 1 

494 manifest = render_k8s( 

495 agent_ids=roster, server_urls=server_urls, mcp_servers=mcp_servers 

496 ) 

497 else: # docker — guarded by the membership check above 

498 manifest = render_docker(repo_compose_path="docker-compose.yml", agent_ids=roster) 

499 

500 if out_path is not None: 

501 out_path.write_text(manifest, encoding="utf-8") 

502 print(f"wrote {target} manifest to {out_path}") 

503 else:
