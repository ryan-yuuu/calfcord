1"""Shared helper for rendering ``openhands`` :class:`Observation` objects 

2as a single LLM-facing string. 

3 

4Every builtin tool that delegates to an openhands executor receives an 

5:class:`Observation` with two fields we care about for the calfkit 

6``return type=str`` contract: 

7 

8* ``is_error: bool`` — true when the upstream tool errored (file not 

9 found, multi-occurrence on str_replace, permission denied, etc.). 

10* ``content: Sequence[TextContent | ImageContent]`` — the human-readable 

11 body, possibly multi-part. Builtin tools today never emit 

12 :class:`ImageContent`, so we walk for ``.text`` only. 

13 

14Without this helper, a tool wrapper that simply concatenates 

15``content[*].text`` returns text indistinguishable from a successful 

16call. The LLM cannot reliably detect failure — and the existing 

17``private_chat`` convention is "errors are returned as strings starting 

18with ``error:``" so the LLM can adapt without triggering retry-with- 

19feedback. 

20 

21:func:`flatten_observation_text` enforces that convention by checking 

22``is_error`` and prefixing accordingly. Use this in every wrapper that 

23delegates to an openhands executor. 

24""" 

25 

26from __future__ import annotations 

27 

28_ERROR_PREFIX = "error: " 

29"""Mirrors the convention in 

30:mod:`calfcord.tools.builtin.private_chat` — recoverable failures 

31return strings starting with ``error:`` so the calling LLM can read the 

32discriminator without parsing structured fields.""" 

33 

34 

35def flatten_observation_text(obs: object) -> str: 

36 """Render an openhands :class:`Observation` as a single LLM-facing string. 

37 

38 Args: 

39 obs: An openhands ``Observation`` (Pydantic model with ``content`` 

40 and ``is_error`` fields). Typed as ``object`` to keep this 

41 module free of an openhands-sdk import at the top level — 

42 the runtime field-access via ``getattr`` works against any 

43 of openhands' concrete observation types. 

44 

45 Returns: 

46 The concatenated ``.text`` of every ``TextContent`` in 

47 ``obs.content``, joined by blank lines. When ``obs.is_error`` is 

48 true, the result is prefixed with ``"error: "`` so the LLM can 

49 distinguish failure from success. Empty content (rare; can 

50 happen for no-op upstream paths) becomes ``"(no output)"``. 

51 """ 

52 parts: list[str] = [] 

53 for item in getattr(obs, "content", ()) or (): 

54 text = getattr(item, "text", None) 

55 if text: 

56 parts.append(text) 

57 body = "\n\n".join(parts) if parts else "(no output)" 

58 if getattr(obs, "is_error", False): 

59 return _ERROR_PREFIX + body 

60 return body
