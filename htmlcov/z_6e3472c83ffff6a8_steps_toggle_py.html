1"""The on-demand step-transcript view button (Phase 3). 

2 

3On the terminal hop the outbox consumer (the SOLE transcript writer) 

4attaches a single secondary button to the agent's final reply *when the 

5turn used tools* (see 

6``docs/design/step-transcripts-and-live-streaming-plan.md`` §5, §7.5, 

7§8). Clicking that button shows the turn's steps as an **ephemeral 

8message** — visible only to the clicker — containing the tool calls and 

9intermediate text the agent produced. The agent's reply is **never** 

10edited: there is no expand/collapse state, no content sentinel, and no 

112000-char surgery on the reply itself. The whole flow stays inside the 

12bridge process: the callback reads the persisted ``delta_json`` from the 

13bridge-local 

14:class:`~calfcord.bridge.transcripts.TranscriptStore`, 

15renders it, and sends it back to the clicker as a private followup. 

16 

17**One-way action, not a toggle.** The button is a *persistent* view 

18(``timeout=None``) with a single static ``custom_id`` 

19(:data:`_TOGGLE_CUSTOM_ID`), registered once via ``add_view`` on the 

20gateway client. Because it never mutates the reply, every click is 

21independent — the user can view the steps as many times as they like, 

22each producing a fresh ephemeral message only they can see. 

23 

24**Why defer ``thinking=True, ephemeral=True``.** The callback 

25:meth:`StepsToggleView._show_steps`'s first action is 

26``interaction.response.defer(thinking=True, ephemeral=True)``. A 

27*component* interaction needs BOTH flags to land an ephemeral followup: 

28``defer(ephemeral=True)`` alone falls through to the invisible 

29``deferred_message_update`` and the ephemeral flag is IGNORED (verified 

30in discord.py 2.7.1). Deferring acknowledges the click immediately and 

31removes Discord's 3-second interaction deadline from the DB read + render 

32that follows; the result is then delivered via 

33:meth:`Interaction.followup.send` with ``ephemeral=True``. 

34 

35**Long transcripts attach a file.** When the rendered steps exceed 

36Discord's 2000-char message cap, the FULL transcript is attached as a 

37``steps.md`` file instead of being inlined — nothing is truncated. 

38"""

90def render_steps(delta_json: str) -> tuple[str, int]: 

91 """Render a persisted ``delta_json`` blob into a full steps block. 

92 

93 Deserializes the blob with pydantic-ai's ``ModelMessagesTypeAdapter``, 

94 runs the shared 

95 :func:`~calfcord.bridge.steps._render_tree_blocks` to get one 

96 rendered string per visual block (a prose block, or a ``● tool(args)`` / 

97 ``⎿ result`` tree block), and joins them with a blank line. The text is 

98 returned in full — NO truncation; the caller decides whether it fits an 

99 inline message or must be attached as a file. 

100 

101 Args: 

102 delta_json: The serialized structured slice of a turn's 

103 ``message_history`` (``ModelMessagesTypeAdapter.dump_json`` of 

104 ``message_history[initial_len:-1]``). 

105 

106 Returns: 

107 ``(text, count)`` where ``text`` is the full joined render and 

108 ``count`` is the number of rendered step parts. 

109 """ 

110 # Imported lazily so the module-import cost (and the vendored 

111 # pydantic-ai type-adapter construction) is paid only when a toggle is 

112 # actually built or clicked, not at bridge import time. 

113 from calfkit._vendor.pydantic_ai.messages import ModelMessagesTypeAdapter 

114 

115 messages = ModelMessagesTypeAdapter.validate_json(delta_json) 

116 rendered = _render_tree_blocks(messages) 

117 return "\n\n".join(rendered), len(rendered)

120class StepsToggleView(discord.ui.View): 

121 """Persistent view backing the on-demand step-transcript button. 

122 

123 One instance is registered on the gateway client via 

124 ``client.add_view`` in ``_on_ready``; that single registration handles 

125 every click carrying :data:`_TOGGLE_CUSTOM_ID`, on any reply message. 

126 The throwaway buttons the outbox attaches to individual messages only 

127 emit the component JSON. 

128 

129 Holds a reference to the bridge-local transcript store so the callback 

130 can read the turn's persisted steps by the clicked message's id. 

131 """

136 super().__init__(timeout=None) 

137 self._store = store

144 async def _show_steps(self, interaction: discord.Interaction, _button: discord.ui.Button[Any]) -> None: 

145 """Show the clicked reply's steps as an ephemeral message. 

146 

147 Algorithm: 

148 

149 1. ``interaction.response.defer(thinking=True, ephemeral=True)`` 

150 FIRST. A *component* interaction needs BOTH ``thinking=True`` 

151 AND ``ephemeral=True`` — ``defer(ephemeral=True)`` alone falls 

152 through to the invisible ``deferred_message_update`` and the 

153 ephemeral flag is IGNORED (verified in discord.py 2.7.1). This 

154 acknowledges the click and lifts the 3-second deadline off the 

155 DB read below. 

156 2. Look up the transcript row by the clicked message's id. The read 

157 is GUARDED: it is a live aiosqlite call (a dedicated-thread DB 

158 query) that can raise on disk I/O, a malformed WAL, or a lock 

159 timeout — none of which is a :class:`discord.HTTPException`, so 

160 without the guard it would escape AFTER the defer and hang the 

161 spinner. On a read error we log + send an error followup. A clean 

162 miss (pruned / pre-restart / never-written / disabled store) 

163 sends the ephemeral "not available for this response" followup 

164 and stops. 

165 3. Render the FULL steps text. When it fits the Discord message 

166 cap, send it inline as an ephemeral followup; otherwise attach 

167 the full transcript as a ``steps.md`` file (nothing truncated). 

168 

169 Every followup is sent via :meth:`_safe_followup`, which swallows 

170 :class:`discord.HTTPException` — a failed send must not raise out 

171 of the component callback (which would surface as an "interaction 

172 failed" to the user and a noisy traceback). 

173 """ 

174 # 1) Ephemeral defer. CRITICAL: a COMPONENT interaction needs BOTH 

175 # thinking=True AND ephemeral=True — defer(ephemeral=True) alone 

176 # falls through to the invisible deferred_message_update and the 

177 # ephemeral flag is IGNORED (verified in discord.py 2.7.1). 

178 try: 

179 await interaction.response.defer(thinking=True, ephemeral=True) 

180 except discord.HTTPException: 

181 logger.warning("steps view: defer failed message_id=%s", getattr(interaction.message, "id", None)) 

182 return 

183 

184 message = interaction.message 

185 if message is None: 

186 # Post-defer: the ephemeral "thinking" spinner is already 

187 # showing, so we MUST land a followup rather than returning 

188 # silently — a bare return leaves the spinner hung forever. 

189 logger.debug("steps view: interaction carried no message; sending an unavailable followup") 

190 await self._safe_followup(interaction, content="Step details are unavailable for this message.") 

191 return 

192 

193 # The store read is the one DB call between the defer and a 

194 # followup; guard it like render_steps below. aiosqlite runs on a 

195 # dedicated thread and can raise (sqlite3.OperationalError on disk 

196 # I/O, a malformed WAL, or a lock beyond busy_timeout) — none of 

197 # which is a discord.HTTPException, so an unguarded raise would 

198 # escape the callback AFTER the defer and hang the ephemeral 

199 # "thinking" spinner until the interaction token expires (~15 min). 

200 # except Exception (not BaseException) so a shutdown CancelledError 

201 # still propagates. 

202 try: 

203 row = await self._store.get_by_final_message_id(str(message.id)) 

204 except Exception: 

205 logger.exception( 

206 "steps view: store read failed message_id=%s; " 

207 "sending an error followup instead of hanging the interaction", 

208 message.id, 

209 ) 

210 await self._safe_followup(interaction, content="Could not load the steps for this response.") 

211 return 

212 if row is None: 

213 # A clean miss: the row was pruned, predates a restart, was 

214 # never written (write-after-post race / swallowed write 

215 # failure), or the store is a disabled NullTranscriptStore. 

216 # Worded so it doesn't claim the data "was" there and vanished. 

217 await self._safe_followup( 

218 interaction, 

219 content="Step details aren't available for this response (it may still be saving, or has expired).", 

220 ) 

221 return 

222 

223 # render_steps can raise: ModelMessagesTypeAdapter.validate_json 

224 # blows up on a corrupt blob, and _render_tree_blocks' 

225 # ToolCallPart.args_as_json_str blows up on malformed tool-call 

226 # args. Either would otherwise escape the callback AFTER the defer, 

227 # hanging the ephemeral spinner. Mirror the outbox's 

228 # _render_step_count guard: log + send an error followup instead. 

229 try: 

230 text, _count = render_steps(row.delta_json) 

231 except Exception: 

232 logger.exception( 

233 "steps view: failed to render stored delta message_id=%s; " 

234 "sending an error followup instead of hanging the interaction", 

235 message.id, 

236 ) 

237 await self._safe_followup(interaction, content="Could not render the steps for this response.") 

238 return 

239 if not text: 

240 await self._safe_followup(interaction, content="This response recorded no steps.") 

241 return 

242 

243 if len(text) <= _DISCORD_MESSAGE_LIMIT: 

244 await self._safe_followup(interaction, content=text) 

245 else: 

246 # Too long to inline → attach the FULL transcript as a file 

247 # (nothing truncated). 

248 f = discord.File(io.BytesIO(text.encode("utf-8")), filename="steps.md") 

249 await self._safe_followup(interaction, file=f)

251 async def _safe_followup( 

252 self, 

253 interaction: discord.Interaction, 

254 *, 

255 content: str = discord.utils.MISSING, 

256 file: discord.File = discord.utils.MISSING, 

257 ) -> None: 

258 """Send an ephemeral followup, swallowing any Discord error. 

259 

260 The followup carries either ``content`` or ``file`` (both default 

261 to :data:`discord.utils.MISSING` so an unsupplied argument is 

262 omitted from the underlying ``followup.send`` call). A failed send 

263 is logged and swallowed — it must never raise out of the component 

264 callback. 

265 """ 

266 try: 

267 await interaction.followup.send(content=content, file=file, ephemeral=True) 

268 except discord.HTTPException: 

269 # Log whether a file rode the send so an oversized-steps.md 413 

270 # (file-only path) is distinguishable from a content-send failure. 

271 logger.warning( 

272 "steps view: followup.send failed message_id=%s file=%s", 

273 getattr(interaction.message, "id", None), 

274 file is not discord.utils.MISSING, 

275 )
