1"""Provider helpers shared by the interactive setup wizards. 

2 

3The ``calfcord init`` flow (and the Round-2 per-agent wizards built on top of 

4it) need three provider-specific operations that all touch heavy, optional 

5dependencies — the OpenAI / Anthropic SDKs, the live Codex catalog, and the 

6ChatGPT-subscription OAuth machinery. Putting them here, behind a small 

7synchronous API, keeps every one of those imports *lazy and function-local* so: 

8 

9* merely importing this module (which the argparse entry point and the test 

10 suite both do at startup) never pulls in an SDK, opens a network socket, or 

11 needs a TTY; 

12* the wizard callers depend only on :class:`~calfcord.cli._prompts.Prompter` 

13 and :class:`~calfcord.cli._prompts.Choice` — they never ``import openai`` / 

14 ``anthropic`` / ``openhands`` themselves, so a host missing one provider's 

15 SDK can still run the wizard for another provider. 

16 

17The two design rules the wizards rely on: 

18 

19* **Always select, never free-text a model.** A live, filtered model list is 

20 fetched per provider; a curated fallback is offered only when the live fetch 

21 fails, so an operator can never type a slug that the provider will reject. 

22* **Never abort the wizard on an auth/network hiccup.** A failed live fetch 

23 degrades to the fallback list with a warning; a failed Codex login prints a 

24 resume hint and returns. The wizard must always reach the point where it 

25 writes config. 

26"""

142def list_models(provider: str, *, api_key: str | None) -> list[Choice]: 

143 """Return the live, selectable models for ``provider`` as :class:`Choice` rows. 

144 

145 Each row's ``value`` is the model id the agent's ``model:`` field will carry; 

146 ``label`` is what the operator sees. The SDK / catalog for each provider is 

147 imported lazily inside its branch so importing this module needs no SDK, TTY, 

148 or network. 

149 

150 Filtering / ordering per provider: 

151 

152 * ``anthropic`` — every model the SDK lists (all are Claude chat models), in 

153 API order, labelled ``"<id> (<display name>)"``. 

154 * ``openai`` — only chat models (see :func:`_is_openai_chat_model`), sorted 

155 by id. If the filter removes everything (an unexpected catalog shape) but 

156 the API *did* return models, fall back to all ids rather than an empty 

157 list, so the wizard still offers a choice. 

158 * ``openai-codex`` — the user-selectable slugs from the live ``models.json`` 

159 catalog (no API key needed). 

160 

161 Raises: 

162 ModelAuthError: when the provider SDK rejected the API key, so callers 

163 can warn the operator loudly that the agent won't work as configured. 

164 ModelListError: on any other underlying fetch failure (network, SDK), so 

165 callers handle one base exception type regardless of provider. 

166 """ 

167 try: 

168 if provider == "anthropic": 

169 import anthropic 

170 

171 page = anthropic.Anthropic(api_key=api_key).models.list() 

172 return [Choice(m.id, f"{m.id} ({m.display_name})") for m in page] 

173 

174 if provider == "openai": 

175 import openai 

176 

177 ids = [m.id for m in openai.OpenAI(api_key=api_key).models.list()] 

178 chat_ids = sorted(i for i in ids if _is_openai_chat_model(i)) 

179 # An empty result after filtering a non-empty catalog means the 

180 # account's model namespace doesn't match our heuristics — offer 

181 # everything rather than block the wizard with no choices. 

182 if not chat_ids and ids: 

183 chat_ids = sorted(ids) 

184 return [Choice(i, i) for i in chat_ids] 

185 

186 if provider == "openai-codex": 

187 slugs = asyncio.run(_load_codex_slugs()) 

188 return [Choice(s, s) for s in slugs] 

189 except Exception as exc: 

190 # Broad on purpose: each provider branch can fail in a different way 

191 # (httpx errors, auth errors, SDK-internal errors) and the callers want 

192 # one exception type to fall back on. A *rejected key* is singled out 

193 # first (as ModelAuthError) because it means the agent won't work at 

194 # runtime — the caller must warn louder than for a transient network 

195 # blip. The SDK imports here are lazy and per-provider so a host missing 

196 # one provider's SDK never trips over it when failing on another. 

197 if _is_auth_error(provider, exc): 

198 raise ModelAuthError(f"API key for {provider!r} was rejected: {exc}") from exc 

199 raise ModelListError(f"could not fetch models for {provider!r}: {exc}") from exc 

200 

201 raise ModelListError(f"unknown provider {provider!r}")

227def _build_fallback_models() -> dict[str, list[str]]: 

228 """Curated last-resort model lists, used only when :func:`list_models` raises. 

229 

230 The flagship of each key-based provider is derived from 

231 :data:`calfcord.agents.factory._PROVIDER_DEFAULT_MODELS` so it tracks the 

232 factory's default and can't silently drift. Deliberately minimal — a 

233 flagship plus a cheap tier per provider — because this list is a degraded 

234 fallback, not the catalog. ``openai-codex`` has no static flagship in the 

235 factory (it resolves live), so it carries a single known-good mini slug. 

236 

237 The ``factory`` import is function-local on purpose: it transitively pulls 

238 in calfkit's provider machinery (and the OpenAI/Anthropic SDK modules), so 

239 deferring it keeps merely importing *this* module SDK-free. Callers reach the 

240 mapping through :func:`fallback_models`, which caches the result of this 

241 builder in :data:`_FALLBACK_MODELS_CACHE`. 

242 

243 Each ``or "<slug>"`` arm is an unreachable belt-and-suspenders default: the 

244 factory constants are non-empty for the key-based providers today, so the 

245 left operand always wins. Keeping the literals means a future ``None`` in the 

246 factory still degrades to a known-good slug rather than crashing the wizard. 

247 """ 

248 from calfcord.agents.factory import _PROVIDER_DEFAULT_MODELS 

249 

250 return { 

251 "anthropic": [_PROVIDER_DEFAULT_MODELS["anthropic"] or "claude-sonnet-4-5", "claude-haiku-4-5"], 

252 "openai": [_PROVIDER_DEFAULT_MODELS["openai"] or "gpt-5-mini", "gpt-5-nano"], 

253 "openai-codex": ["gpt-5-codex-mini"], 

254 } 

255 

256 

257# Lazily-populated cache for :func:`fallback_models`. ``None`` until first use so 

258# importing this module never imports the factory (which would eagerly load the 

259# provider SDKs). Callers/tests that need the mapping go through 

260# :func:`fallback_models`; the name below is kept populated as a convenience view. 

261_FALLBACK_MODELS_CACHE: dict[str, list[str]] | None = None

265 """Return the curated fallback mapping, building (and caching) it on first call. 

266 

267 Lazy so that importing :mod:`calfcord.cli._providers` does not import 

268 :mod:`calfcord.agents.factory` (and through it the provider SDKs). The result

309def pick_model( 

310 prompter: Prompter, 

311 provider: str, 

312 *, 

313 api_key: str | None, 

314 cheap: bool = False, 

315 current: str | None = None, 

316) -> str: 

317 """Prompt the operator to *select* (never free-text) a model for ``provider``. 

318 

319 Tries :func:`list_models`; on failure it falls back to the curated 

320 :func:`fallback_models` list for the provider so the wizard keeps working 

321 offline / with a bad key. A :class:`ModelAuthError` (rejected key) is caught 

322 *first* and surfaced LOUDLY — a curated fallback still lets the operator 

323 finish setup, but the agent won't actually work until the key is fixed, so 

324 the generic offline-ish warning would understate it. The pre-selected default 

325 is chosen by :func:`_recommended_default` (cheap tier when ``cheap`` is set, 

326 otherwise the flagship, then the operator's ``current`` value, then the 

327 first row) and is always one of the offered values. 

328 

329 Returns the selected model id. 

330 """ 

331 try: 

332 choices = list_models(provider, api_key=api_key) 

333 except ModelAuthError: 

334 print( 

335 f"warning: the API key for {provider} was REJECTED — the agent won't work " 

336 f"until you fix it; re-run 'calfcord init' to re-enter it." 

337 ) 

338 choices = _fallback_choices(provider) 

339 except ModelListError as exc: 

340 print(f"warning: couldn't fetch live models for {provider} ({exc}); choose from known models") 

341 choices = _fallback_choices(provider) 

342 

343 if not choices: 

344 # A fetch that SUCCEEDS but returns nothing (a Codex catalog with only 

345 # hidden models, an account that lists zero) would otherwise hand the 

346 # prompter an empty choice list and crash it ("choices cannot be empty"), 

347 # aborting the wizard — the one thing this module promises never to do. 

348 # Treat empty-on-success like a failed fetch and offer the curated list. 

349 print(f"warning: no models returned for {provider}; choose from known models") 

350 choices = _fallback_choices(provider) 

351 

352 ids = [c.value for c in choices] 

353 if not ids: 353 ↛ 357line 353 didn't jump to line 357 because the condition on line 353 was never true

354 # Curated fallback is empty too — only reachable for an unknown provider 

355 # (the wizard never produces one). Fail with a clear message rather than 

356 # crash the prompter on an empty list. 

357 raise ModelListError(f"no models available for {provider!r} and no curated fallback") 

358 default = _recommended_default(provider, ids, cheap=cheap, current=current) 

359 return prompter.select(f"Model for {provider}?", choices, default=default)

368 prompter: Prompter, provider: str, *, env_path: Path, current: dict[str, str]

370 """Ensure the chosen ``provider`` can authenticate, writing keys to ``env_path``. 

371 

372 For key-based providers (``anthropic`` / ``openai``) it prompts for the 

373 provider's API-key var, masking the input and writing it only when a value

377 a no-op. 

378

380 entered value, or the existing one when the operator kept it), or ``None``

388 if value:

390 return value 

391 return current.get(key_var)

398def configure_provider( 

399 prompter: Prompter, 

400 *, 

401 env_path: Path, 

402 current: dict[str, str], 

403 default_provider: str | None = None, 

404 cheap: bool = False, 

405 current_model: str | None = None, 

406) -> ProviderModel: 

407 """Run the provider sub-flow shared by every wizard and return ``(provider, model)``. 

408 

409 Selects a provider (``default_provider`` pre-selects the menu — e.g. the 

410 router wizard defaults to the agent provider), ensures its credentials 

411 (key prompt or inline Codex OAuth), then selects a model from the live list. 

412 ``cheap`` biases the model default toward a fast/cheap model (the router); 

413 ``current_model`` pre-selects the operator's existing choice on a re-run. 

414 The caller decides where to persist the result (an agent ``.md`` vs the 

415 ``CALFKIT_ROUTER_*`` env vars), so this never writes the provider/model 

416 itself — only the credential side effect of :func:`ensure_credentials`. 

417 """ 

418 provider = prompter.select("Model provider?", PROVIDERS, default=default_provider or "anthropic") 

419 api_key = ensure_credentials(prompter, provider, env_path=env_path, current=current) 

420 # A key-based provider with no effective key (operator skipped a never-set 

421 # field) will fail at runtime, so say so now rather than letting the curated 

422 # fallback model list imply a working setup. 

423 if provider in _PROVIDER_KEY_VAR and not api_key: 

424 print( 

425 f"warning: no {_PROVIDER_KEY_VAR[provider]} is set — {provider} won't work " 

426 f"until you add one (re-run 'calfcord init')." 

427 ) 

428 model = pick_model(prompter, provider, api_key=api_key, cheap=cheap, current=current_model) 

429 return ProviderModel(provider, model)

432def _codex_login() -> None: 

433 """Run the inline ChatGPT-subscription (Codex) OAuth flow. 

434 

435 Steps: 

436 

437 1. If cached credentials are still valid (or can be silently refreshed), 

438 report that and stop — re-running the wizard must not force a re-login. 

439 2. Otherwise log in straight away — picking Codex is itself the consent, so 

440 there is no extra yes/no prompt. We always use the **device-code** flow: 

441 it prints a URL + one-time code to open on any device and polls for 

442 completion, with no localhost OAuth callback, so it works identically on a 

443 local desktop and over SSH / a headless VM (the browser flow binds a 

444 localhost callback the operator's machine can't reach over SSH). 

445 

446 Any failure (network, OAuth error) is *caught*, surfaced as a warning with a 

447 resume hint (``calfcord calfkit-auth codex login``), and swallowed — the wizard 

448 must still proceed to write the rest of the config. Auth is never the thing 

449 that aborts setup. The OAuth machinery is imported lazily so this module 

450 stays SDK-free at import time. 

451 """ 

452 from openhands.sdk.llm.auth import OpenAISubscriptionAuth 

453 

454 from calfcord.providers.codex.token_store import get_credential_store 

455 

456 store = get_credential_store() 

457 auth = OpenAISubscriptionAuth(credential_store=store) 

458 

459 try: 

460 creds = asyncio.run(auth.refresh_if_needed()) 

461 except Exception: 

462 # Any refresh failure (expired token, no cache, network) just means the 

463 # operator isn't logged in yet — fall through to the interactive login. 

464 creds = None 

465 if creds is not None: 

466 print("Already authenticated with ChatGPT.") 

467 return 

468 

469 print("Logging in to ChatGPT — open the URL below and enter the code:") 

470 try: 

471 asyncio.run(auth.login(auth_method="device_code", open_browser=False)) 

472 except Exception as exc: 

473 # Broad on purpose: a declined/failed/aborted OAuth login must never 

474 # tear down the wizard — warn with a resume hint and let setup finish. 

475 print( 

476 f"warning: Codex login did not complete ({exc}). " 

477 "You can finish it later with: calfcord calfkit-auth codex login" 

478 ) 

479 return 

480 print("Logged in to ChatGPT.")
